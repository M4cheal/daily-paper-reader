## Abstract
Computer Vision (CV) has yet to fully achieve the zero-shot task generalization observed in Natural Language Processing (NLP), despite following many of the milestones established in NLP, such as large transformer models, extensive pre-training, and the auto-regression paradigm, among others. In this paper, we rethink the reality that CV adopts discrete and terminological task definitions (e.g., "image segmentation"), and conjecture it is a key barrier that hampers zero-shot task generalization. Our hypothesis is that without truly understanding previously-seen tasks—due to these terminological definitions—deep models struggle to generalize to novel tasks. To verify this, we introduce Explanatory Instructions, which provide an intuitive way to define CV task objectives through detailed linguistic transformations from input images to outputs. We create a large-scale dataset comprising 12 million "image input $\to$ explanatory instruction $\to$ output" triplets, and train an auto-regressive-based vision-language model (AR-based VLM) that takes both images and explanatory instructions as input. By learning to follow these instructions, the AR-based VLM achieves instruction-level zero-shot capabilities for previously-seen tasks and demonstrates strong zero-shot generalization for unseen CV tasks.  Code and dataset will be open-sourced.

## 3. 实验设计

### 实验设置
- **数据集**：DECVT包含**术语化视觉任务组件**（从AllWeather、NYU-Rain、D-HAZE、LVIS、ADE20K等公开数据集收集或生成）和**解释性视觉任务组件**（从MagicBrush、InstructPix2Pix、HQ-Edit等编辑数据集及搜索引擎收集的图像对构建）。
- **基准对比**：对比了ControlNet、T2I-Adapter、LDM、LaMa、DeepFill v2、MaskGIT、OmniGen、PixWizard、Lumina-mGPT等任务特定或视觉通用模型。
- **评估任务**：
  - **生成任务**：Canny-to-Image、HED-to-Image、Inpainting、Outpainting
  - **密集预测任务**：语义分割（ADE20K）、深度估计（NYU-v2）、法线估计（NYU-v2）
  - **低层视觉任务**：去雨（Rain100L）、去雾（SOTS）、图像恢复（UDC）、低光增强（LOLv2）
  - **图像编辑**：MagicBrush test set、Emu Edit test set
- **评估指标**：FID、CLIP-Score、SSIM、F1、LPIPS、IS、PSNR、mIoU、RMSE、Mean Angle Error等。

### 两大实验场景
1. **指令级零样本**：在全数据集训练后，用**未见过的解释性指令**测试（如图5，用户自定义编辑指令）。
2. **任务级零样本**：从训练集中**排除某些任务**（如Canny-to-Image、Depth-to-Image、HED-to-Image等），然后用对应任务的解释性指令测试模型是否能完成这些从未见过的任务（图6-8）。

## 4. 资源与算力

- **全数据集训练**（7B模型）：
  - GPU：64块A100（40GB）
  - 批次大小：每GPU 8（全局512）
  - 训练轮数：2个epoch（约47k次迭代）
  - 总GPU时间：**5,400 GPU小时**
- **快速验证（任务级零样本）**：
  - GPU：8块A100
  - 批次大小：128（全局1024）
  - 训练轮数：2个epoch（约46k次迭代）
  - 总GPU时间：**1,340 GPU小时**
- 初始化模型：Lumina-mGPT-7B-768（对于全数据）和Lumina-mGPT-7B-768-Omni（对于任务级零样本）。
- 图像分辨率：448×448。

## 5. 实验数量与充分性

- 实验**覆盖较广**：包括生成任务（4种）、密集预测（3种）、低层任务（4种）、图像编辑（2个benchmark）——共计至少13个评估场景。
- **指令级零样本**：展示了多种未见指令下的成功案例（图5）。
- **任务级零样本**：在3种未见任务（HED-to-Image、Canny-to-Image、Depth-to-Image）上验证了可行性（图6-8），同时在Outpainting、Inpainting、Low-light Enhancement、Deblurring等任务上也展示了能力（附录C）。
- **定量结果**：给出了详细的数值表（表1、表2、表7），并标注了哪些是任务级零样本（T. Z.-s.）或指令级零样本（I. Z.-s.）。
- **公平性**：对比基线中，其他方法通常使用**任务特定的固定指令或图像标注**，而本模型使用**从未见过的解释性指令**进行推理，因此对比是严格且设定更苛刻的。
- **不足之处**：未包含专门的消融实验（如不同指令长度、不同模型规模的影响），也未对数据集各组件贡献进行逐一分析。同时，任务级零样本仅在**生成任务**上成功，在反向任务（如Image-to-Canny）上失败。

## 6. 论文的主要结论与发现

1. **解释性指令能够使VLM实现指令级零样本泛化**：模型可以理解从未见过的自然语言描述，并完成相应的视觉任务（如图5中的创意编辑）。
2. **在生成任务上展示了任务级零样本能力**：对于训练中从未出现过的Canny-to-Image、HED-to-Image、Depth-to-Image等任务，仅通过解释性指令引导，模型就能生成合理输出。
3. **优于基线Lumina-mGPT**：在几乎所有定量指标上，微调后的模型显著优于原始Lumina-mGPT（如Canny-to-Image FID从61.65降至26.93，HED-to-Image FID从69.49降至15.91）。
4. **验证了假设：离散任务定义阻碍泛化**：当用解释性指令代替术语化标签时，模型能够更好地跨任务泛化。
5. **识别了当前局限性**：任务级零样本不适用于某些反向任务（如图像到边缘图），原因在于预训练图像tokenizer与文本模态对齐不足。

## 7. 优点

- **创新性**：首次提出用**解释性指令**打破传统视觉任务边界，为CV通用智能提供新范式。
- **数据规模与多样性**：DECVT包含1200万三元组，涵盖多种低层、高层、编辑任务，并且包含双向变换（正向和反向），极大丰富了任务表达。
- **实验设计严谨**：明确区分指令级零样本和任务级零样本，并设置了严格对比（其他方法使用固定指令，本方法使用未见指令）。
- **可解释性**：解释性指令本身具有人类可读性，可以揭示模型如何理解任务目标。
- **开源承诺**：代码和数据集将开放，促进后续研究。

## 8. 不足与局限

- **性能差距**：与任务特定模型（如ControlNet、PromptIR）以及一些通用模型（如PixWizard）相比，零样本性能仍有较大差距。
- **任务级零样本有限**：仅成功用于**生成任务**，在**反向预测任务**（如Image-to-Canny、Image-to-Depth）上失败。作者认为根本原因是图像tokenizer与文本模态对齐不足。
- **模型稳定性**：生成结果对top-k等超参数敏感，且存在物体细节丢失、光照控制不精确等问题（附录C已指出）。
- **数据噪声**：GPT-4o生成的解释性指令可能存在描述偏差或不准确，影响模型训练。
- **指令复杂度权衡**：过复杂的指令可能降低模型理解效果，存在最佳复杂度阈值。
- **资源消耗大**：训练需要大量GPU算力（5400 GPU小时），对一般研究团队门槛较高。
- **消融实验缺失**：未分析不同数据组分、指令长度、模型规模等因素的影响。
- **未扩展至视频/3D**：论文仅处理图像对，视频和3D数据的零样本泛化尚待探索。
