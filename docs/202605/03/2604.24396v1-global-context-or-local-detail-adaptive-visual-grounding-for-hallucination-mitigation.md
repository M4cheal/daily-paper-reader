## 摘要
视觉语言模型（VLMs）经常受到物体幻觉的困扰——即生成与视觉现实相矛盾的内容——这是由于过度依赖语言先验所致。我们引入了正负解码（Positive-and-Negative Decoding, PND），这是一种无需训练的推理框架，通过直接干预解码过程来增强视觉忠实度。PND 的灵感源于我们的一个关键发现，即 VLM 中存在严重的注意力缺失，实验表明视觉特征的权重被低估。我们的框架通过双路径对比来纠正这一问题：正向路径利用多层注意力放大显著的视觉证据，以鼓励忠实的描述，直接抵消注意力缺失；同时，负向路径识别并降级核心物体的特征以创建强有力的反事实，从而惩罚缺乏根据、由先验主导的生成。通过在每一步对比模型在这两个视角下的输出，PND 引导生成不仅在语言上合理、而且在视觉上真实的文本。在 POPE、MME 和 CHAIR 等基准测试上的广泛实验表明，PND 实现了最先进的性能，准确率提升高达 6.5%，在显著减少物体幻觉的同时增强了描述细节，且无需任何模型重训练。该方法在 LLaVA、InstructBLIP、InternVL 和 Qwen-VL 等多种 VLM 架构上均具有良好的泛化性。

## Abstract
Vision-Language Models (VLMs) are frequently undermined by object hallucination--generating content that contradicts visual reality--due to an over-reliance on linguistic priors. We introduce Positive-and-Negative Decoding (PND), a training-free inference framework that intervenes directly in the decoding process to enforce visual fidelity. PND is motivated by our key finding of a critical attention deficit in VLMs, where visual features are empirically under-weighted. Our framework corrects this via a dual-path contrast: The positive path amplifies salient visual evidence using multi-layer attention to encourage faithful descriptions, directly counteracting the attention deficit. Simultaneously, the negative path identifies and degrades the core object's features to create a strong counterfactual, which penalizes ungrounded, prior-dominant generation. By contrasting the model's outputs from these two perspectives at each step, PND steers generation towards text that is not just linguistically probable, but visually factual. Extensive experiments on benchmarks like POPE, MME, and CHAIR show that PND achieves state-of-the-art performance with up to 6.5% accuracy improvement, substantially reducing object hallucination while also enhancing descriptive detail--all without requiring any model retraining. The method generalizes effectively across diverse VLM architectures including LLaVA, InstructBLIP, InternVL, and Qwen-VL.

### 1. 核心问题与整体含义
*   **研究背景**：大型视觉语言模型（LVLMs）虽然在多模态推理上表现优异，但仍面临严重的“物体存在幻觉”问题（即模型自信地宣称图像中不存在的物体）。
*   **核心问题**：现有的“图像思考”（Think-with-Images, TwI）方法（通过生成缩放裁剪图或高亮图来辅助推理）存在两个瓶颈：
    1.  **粒度-上下文权衡**：缩放（Zoom-in）能看清局部细节但破坏了全局拓扑关系；高亮（Highlight）保留了全局背景但缺乏精细分辨率。
    2.  **工具过度信任**：模型往往盲目信任外部感知工具（如检测器）提供的区域，一旦工具误报，模型推理就会被误导。
*   **研究目的**：提出一种自适应的视觉对齐框架，在不增加训练成本的前提下，通过“冲突驱动”的机制来平衡全局与局部信息，缓解幻觉。

### 2. 方法论：Active-Look 框架
*   **核心思想**：利用两个异构的视觉专家工具（GroundingDINO 和 OWLv2）并行运行，将它们的“意见分歧”作为不确定性的代理指标，仅对有争议的区域投入额外的计算资源进行验证。
*   **算法流程**：
    1.  **假设驱动感知**：从用户查询中提取目标物体，利用双专家工具生成候选区域提议。
    2.  **共识仲裁（Consensus Arbitration）**：通过计算 IoU（交并比）将提议分为“信任集”（两者一致）和“怀疑集”（两者分歧）。
    3.  **冲突感知混合渲染**：
        *   **全局高亮**：对所有提议区域进行高亮处理，保留全局拓扑。
        *   **选择性缩放**：仅对“怀疑集”中的区域进行局部缩放裁剪，以获取更高分辨率的细节。
    4.  **多视图推理**：将原始图、高亮图和缩放后的局部图同时输入 LVLM，由模型作为最终裁决者进行推理。

### 3. 实验设计
*   **数据集**：
    *   **POPE (Adversarial)**：专门用于诊断物体存在幻觉的基准。
    *   **MME**：评估存在性、计数、位置和颜色四个维度的幻觉。
    *   **CHAIR**：评估自由格式图像描述（Captioning）中的物体幻觉率。
    *   **TallyQA & GQA (子集)**：用于尺度诊断和复杂计数场景的压力测试。
*   **基准模型（Backbones）**：LLaVA-1.5 (7B/13B), Qwen3-VL (4B/8B/32B), InternVL2 (4B/8B/14B)。
*   **对比方法**：包括原始 Baseline、推理时干预方法（VCD, OPERA, VAF）以及单一操作的 TwI 变体（纯 Highlight 或纯 Zoom-in）。

### 4. 资源与算力
*   **硬件环境**：论文提到在 **NVIDIA H20 GPU** 上进行测试。
*   **算力开销**：

    *   **训练时长**：该方法为 **Training-free（免训练）**，无需额外的训练时长。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 3 个主流幻觉基准，测试了 3 种不同架构、多个参数规模（从 4B 到 32B）的 LVLM 模型。
*   **消融实验**：详细分析了双专家组合的必要性、混合渲染策略的有效性，以及参数（如缩放因子、IoU 阈值）的敏感性。
*   **客观性**：通过“尺度诊断实验”证明了不同操作在不同物体大小下的表现差异，从而论证了自适应选择的科学性。实验对比了多种 SOTA 方法，结果具有说服力。

### 6. 主要结论与发现
*   **性能提升**：Active-Look 在 POPE 准确率上比 Baseline 提升了约 **4-5%**，在 MME 总分和 CHAIR 幻觉率上均有显著改善。
*   **冲突价值**：单纯增加工具提议（Union）反而可能引入噪声降低性能，而通过“冲突仲裁”识别不确定区域是提升鲁棒性的关键。
*   **尺度敏感性**：缩放对小物体有效，高亮对大物体和关系推理有效，Active-Look 成功结合了两者优点。
*   **模型通用性**：该方法对更强大的模型（如 32B 参数量）依然有显著增益，说明视觉重新感知是解决单次编码瓶颈的有效途径。

### 7. 优点
*   **即插即用**：无需修改模型权重或重新训练，易于部署到现有 LVLM。
*   **机制创新**：引入了类似人类“瞥一眼（Glance）与盯着看（Stare）”的逻辑，通过专家分歧来分配注意力。
*   **鲁棒性强**：通过双专家校验，有效缓解了单一感知工具出错导致的“单点故障”。

### 8. 不足与局限
*   **工具依赖**：系统的上限受限于外部专家工具（GroundingDINO/OWLv2）的召回率，如果两者都漏检，系统无法触发验证。
*   **推理成本**：2.3 倍的延迟和增加的 Token 消耗在实时性要求极高的场景下可能存在压力。
*   **任务覆盖**：目前主要针对物体存在性和属性幻觉，对于复杂的动作理解或时序推理（视频）尚未验证。
