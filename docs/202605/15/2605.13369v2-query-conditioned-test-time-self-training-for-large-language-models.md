## 摘要
大语言模型（LLMs）通常以固定参数部署，其性能往往通过在推理时分配更多计算资源来提升。虽然这种测试时缩放（test-time scaling）可能有效，但它无法纠正模型的错误认知，也无法使模型适应单个查询的具体结构。测试时优化（Test-time optimization）通过在推理期间允许参数更新来解决这一局限性，但现有方法要么依赖外部数据，要么优化缺乏查询特定对齐的通用自监督目标。在这项工作中，我们提出了查询条件化测试时自我训练（QueST），这是一个在推理过程中利用直接源自输入查询的监督信号来调整模型参数的框架。我们的核心见解是，输入查询本身编码了足以构建结构相关“问题-解决方案”对的潜在信号。基于此，QueST 生成此类查询条件化的配对，并将其作为测试时参数高效微调（parameter-efficient fine-tuning）的监督信号。随后，调整后的模型被用于生成最终答案，从而在无需任何外部数据的情况下实现查询特定的自适应。在七个数学推理基准测试和 GPQA-Diamond 科学推理基准测试中，QueST 一贯优于强大的测试时优化基准方法。这些结果表明，查询条件化自我训练是大语言模型测试时自适应的一种有效且实用的范式。代码可在 https://chssong.github.io/Query-Conditioned-TTST/ 获取。

## Abstract
Large language models (LLMs) are typically deployed with fixed parameters, and their performance is often improved by allocating more computation at inference time. While such test-time scaling can be effective, it cannot correct model misconceptions or adapt the model to the specific structure of an individual query. Test-time optimization addresses this limitation by enabling parameter updates during inference, but existing approaches either rely on external data or optimize generic self-supervised objectives that lack query-specific alignment. In this work, we propose Query-Conditioned Test-Time Self-Training (QueST), a framework that adapts model parameters during inference using supervision derived directly from the input query. Our key insight is that the input query itself encodes latent signals sufficient for constructing structurally related problem--solution pairs. Based on this, QueST generates such query-conditioned pairs and uses them as supervision for parameter-efficient fine-tuning at test time. The adapted model is then used to produce the final answer, enabling query-specific adaptation without any external data. Across seven mathematical reasoning benchmarks and the GPQA-Diamond scientific reasoning benchmark, QueST consistently outperforms strong test-time optimization baselines. These results demonstrate that query-conditioned self-training is an effective and practical paradigm for test-time adaptation in LLMs. Code is available at https://chssong.github.io/Query-Conditioned-TTST/.

*   **研究背景**：现有的测试时优化（TTO）方法存在局限：
    *   **TTT-NN** 等方法依赖外部大规模数据库进行检索，实际应用受限。
    *   **TLM** 等方法通过最小化输入困惑度（Perplexity）进行优化，但这种自监督目标与特定查询所需的复杂推理结构缺乏对齐。
*   **研究意义**：论文提出 **QueST** 框架，旨在不依赖外部数据的前提下，通过挖掘输入查询自身的潜在信号，在推理瞬间实现模型参数的“量身定制”。

### 2. 方法论：QueST 核心思想与技术细节
QueST 的核心流程分为三个阶段：
1.  **查询条件化监督生成（Query-conditioned Supervision Generation）**：
    *   将用户输入的查询 $q$ 作为种子（Seed）。
    *   利用模型自身（作为生成器 $p_\phi$）生成 $N$ 个在概念、假设和推理模式上与 $q$ 结构相关的辅助“问题-解决方案”对 $D(q) = \{(x_i, y_i)\}_{i=1}^N$。
2.  **低秩测试时优化（Low-rank Test-time Optimization）**：
    *   采用 **LoRA**（低秩适配）技术，仅更新一小部分新增参数，保持原始模型参数 $\theta$ 固定。
    *   在生成的辅助数据集 $D(q)$ 上进行 $T$ 步有监督微调（SFT），最小化交叉熵损失。这一步使模型在推理当前查询前，先在“类似问题”上进行了预演。
3.  **最终响应生成**：
    *   使用适配后的模型参数 $\theta'$ 对原始查询 $q$ 进行推理，生成最终答案 $y^*$。

### 6. 主要结论与发现
*   **性能领先**：QueST 在所有数学基准上均显著优于 TENT 和 TLM，平均准确率提升了 **6.44 个百分点**。

*   **错误纠正能力**：定性分析显示，QueST 能通过辅助问题的预演，纠正模型在原始查询中因依赖表面模式而产生的逻辑错误。
*   **泛化能力**：在 GPQA-Diamond 上的成功证明了该方法不仅限于数学，也适用于复杂的科学推理。

### 7. 优点
*   **无需外部数据**：完全自给自足，解决了隐私和检索开销问题。
*   **查询特定性**：相比通用的自监督目标，QueST 生成的监督信号与当前任务高度对齐。
*   **参数高效**：利用 LoRA 实现了推理时的快速参数更新，避免了全量微调的不稳定性。

### 8. 不足与局限
*   **依赖生成质量**：如果模型对原始查询完全无法理解，生成的辅助问题可能存在误导性或质量低下，导致“垃圾进，垃圾出”。
*   **推理延迟**：尽管是轻量级微调，但每条查询都需要进行 $T$ 步梯度更新，这在实时性要求极高的场景下可能存在挑战。
*   **知识不积累**：目前的框架在处理完一个查询后会重置参数，无法实现跨查询的持续学习（虽然作者将其列为未来工作）。
*   **错误传播风险**：如果生成的辅助答案是错误的，模型可能会在测试时强化这些错误逻辑。
