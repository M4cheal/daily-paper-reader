## 摘要
测试时扩展通过增加计算量来探索多个解决方案轨迹，从而提升语言模型的推理能力。其关键挑战在于，在尽量减少推理过程中生成的token总数的同时最大化准确性。最近的PRM引导方法通过评分中间前缀来引导搜索，但大多数方法仅限于前沿：它们只保留当前活跃的前缀，并使用带有噪声的PRM评分不可逆地剪枝或重新采样其余部分。这可能导致过早承诺、多样性崩溃以及丢失那些本可以得出正确续写的中间前缀。我们引入了基于持久历史前缀池的随机回溯，使得测试时计算能够重新访问之前生成的状态，而不仅仅是扩展当前前沿。为了提高效率，我们提出了两种互补机制。子池选择通过随机子池内的Top-N选择来增强贪婪的PRM引导搜索，使历史前缀有机会绕过评分过高的前沿候选。幂回溯顺序蒙特卡洛方法利用幂次PRM评分和混合校正权重，将SMC风格的重新采样扩展到持久池。在数学推理基准测试和不同模型规模上，我们的方法在每token数量上始终实现更高的准确性，并且与强PRM引导基线相比，仅使用一小部分token即可达到相同的准确性水平，这表明持久池随机回溯为改善测试时扩展中的准确性- token权衡提供了一种简单有效的方法。

## Abstract
Test-time scaling improves language model reasoning by spending additional compute to explore multiple solution trajectories. The key challenge is to maximize accuracy while minimizing the total number of generated tokens during reasoning. Recent PRM-guided methods score intermediate prefixes to steer this search, but most are frontier-only: they keep only the current active prefixes and irreversibly prune or resample away the rest using noisy PRM scores. This can cause premature commitment, diversity collapse, and the loss of prefixes that still admit correct continuations. We introduce stochastic backtracking over a persistent pool of historical prefixes, allowing test-time compute to revisit previously generated states instead of only expanding the current frontier. To make this efficient, we propose two complementary mechanisms. Subpool Selection strengthens greedy PRM-guided search by applying Top-N selection within random subpools, giving historical prefixes a chance to bypass over-scored frontier candidates. Power Backtrack Sequential Monte Carlo extends SMC-style resampling to the persistent pool using powered PRM scores and mixture-corrected weights. Across mathematical reasoning benchmarks and model scales, our methods consistently achieve higher accuracy per token count, and the same level of accuracy using only a fraction of the token count in comparison to strong PRM-guided baselines, demonstrating that persistent-pool stochastic backtracking provides a simple and effective way to improve the accuracy-token trade-off in test-time scaling.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有测试时扩展（Test-Time Scaling, TTS）方法在语言模型数学推理中，通常采用“前沿仅存”（frontier-only）搜索策略——只保留当前最新生成的前缀（prefix），并利用过程奖励模型（PRM）的噪声评分进行不可逆剪枝。这种机制会导致**过早承诺**（premature commitment）、**多样性崩溃**（diversity collapse），以及丢失那些虽然当前评分较低但实际仍可导向正确答案的历史前缀。
- **研究动机**：PRM评分存在噪声、局部歧义以及生成器不匹配的问题，而前沿仅存搜索将这些局部错误永久化，严重限制了计算效率与最终准确率。
- **整体含义**：本文提出**持久池随机回溯**（persistent-pool stochastic backtracking）——将历史前缀保留在持久池中，使搜索可以在后续轮次中重新访问这些被废弃的状态，从而在不显著增加内存的前提下改善准确率- token权衡。

## 2. 论文提出的方法论

### 2.1 核心思想
- **持久池**：维护一个线性增长的池子 \(P_t\)，包含所有历史产生的前缀（最多 \(N \cdot t\) 个，\(N\) 为每轮扩展数）。
- **随机回溯**：每轮选择父节点时，不局限于当前前沿，而是可以通过随机机制从持久池中采样，使被低估的历史前缀获得再次扩展的机会。
- **两种具体实例**：
  1. **子池选择（Subpool Selection, SPS）**：在每个搜索轮次，从持久池中随机抽取一个大小为 \(K_t\) 的子池，再在该子池内选择 Top-M 作为父节点。这避免了全局贪婪选择中被高分前沿“阻塞”历史前缀的问题。
  2. **幂回溯顺序蒙特卡洛（Power Backtrack SMC, PB-SMC）**：将 SMC 重采样扩展到持久池。每轮保留 \(N t\) 个历史样本（带权），同时生成 \(N\) 个新子节点，组成联合池。使用权重的混合校正公式（公式4-5），目标分布是幂次 PRM 加权的后验。定理1保证该估计量渐近收敛于目标分布。

### 2.2 关键技术细节
- **主算法（Algorithm 1）**：每轮由选择规则 \(f_t\) 采样 \(M\) 个父节点，扩展生成 \(N\) 个子节点，然后按更新规则 \(G_t\) 更新持久池和权重。
- **SPS**：子池大小 \(K_t = \max\{M, \lfloor \rho_t |P_{t-1}| \rfloor\}\)，其中 \(\rho_t\) 取当前池中 PRM 平均分，自适应调整探索程度。
- **PB-SMC**：采用多重重要性采样（multiple importance sampling）框架，结合新生成分布和历史保留分布，权重 \(\alpha_t\) 与线性衰减的混合调度（\(g_{\max}=1, g_{\min}=0.4\)），功率参数 \(\gamma=9\)。

- 持久池随机回溯（SPS 和 PB-SMC）在所有模型和基准上**显著提高了准确率- token 效率**，在多数情况下以 8 个候选达到或超过 32 个候选的基线。

## 7. 优点

- **方法创新**：首次在 TTS 中系统引入持久历史池和随机回溯，直接应对前沿仅存的致命缺陷。
- **无需训练**：仅修改推理时搜索策略，可即插即用，兼容现有 LLM 和 PRM。
- **内存高效**：持久池线性增长（\(Nt\)），不存储完整搜索树；KV 缓存仅对当前扩展前缀保留，回溯时重新计算。
- **两种互补机制**：SPS 适合贪婪搜索增强，PB-SMC 适合采样式搜索，覆盖不同场景。
- **实验充分**：多模型多基准、消融、缩放、超参数敏感性，结果可靠。
- **理论贡献**：PB-SMC 的权重公式和渐近一致性证明为方法提供了理论基础。

## 8. 不足与局限
