## 摘要
可负担性定位（Affordance grounding）需要识别智能体在开放世界场景中应在何处以及如何进行交互，而这些场景中的可操作区域通常较小、存在遮挡、具有反射性且视觉上存在歧义。因此，近期的系统结合了多种技能（如检测、分割、交互想象），但大多数系统采用固定的流水线进行编排，难以匹配逐实例的难度，对中间错误的针对性恢复能力有限，且无法复用来自重复出现物体的经验。这些失败暴露了一个系统性问题：测试时定位必须获取正确的证据，决定该证据是否足够可靠以进行提交，并在无标签的情况下在受限的推理成本内完成。我们提出了 Affordance Agent Harness，这是一个将异构技能与证据库和成本控制统一起来的闭环运行时，它检索情节记忆以提供重复类别的先验，并采用路由器（Router）自适应地选择技能并参数化。随后，一个针对可负担性的验证器（Verifier）利用自一致性、跨尺度稳定性和证据充分性对提交进行门控，在最终评判器将积累的证据和轨迹融合到预测中之前触发针对性的重试。在多个可负担性基准测试和难度受控子集上的实验表明，该方法比固定流水线基准具有更强的准确度-成本帕累托前沿，在提高定位质量的同时减少了平均技能调用次数和延迟。项目页面：https://tenplusgood.github.io/a-harness-page/。

## Abstract
Affordance grounding requires identifying where and how an agent should interact in open-world scenes, where actionable regions are often small, occluded, reflective, and visually ambiguous. Recent systems therefore combine multiple skills (e.g., detection, segmentation, interaction-imagination), yet most orchestrate them with fixed pipelines that are poorly matched to per-instance difficulty, offer limited targeted recovery from intermediate errors, and fail to reuse experience from recurring objects. These failures expose a systems problem: test-time grounding must acquire the right evidence, decide whether that evidence is reliable enough to commit, and do so under bounded inference cost without access to labels. We propose Affordance Agent Harness, a closed-loop runtime that unifies heterogeneous skills with an evidence store and cost control, retrieves episodic memories to provide priors for recurring categories, and employs a Router to adaptively select and parameterize skills. An affordance-specific Verifier then gates commitments using self-consistency, cross-scale stability, and evidence sufficiency, triggering targeted retries before a final judge fuses accumulated evidence and trajectories into the prediction. Experiments on multiple affordance benchmarks and difficulty-controlled subsets show a stronger accuracy-cost Pareto frontier than fixed-pipeline baselines, improving grounding quality while reducing average skill calls and latency. ### 论文总结：Affordance Agent Harness (A-Harness)

#### 1. 核心问题与整体含义
*   **研究背景**：可负担性接地（Affordance Grounding）旨在识别图像中“何处”以及“如何”与物体交互（如抓取把手、按压按钮）。在开放世界中，这些区域通常极小、被遮挡或具有视觉歧义。
*   **核心痛点**：
    1.  **固定流水线的僵化性**：现有方法通常按预定顺序执行检测、分割等技能，导致简单案例过度处理，复杂案例证据不足。
    2.  **缺乏闭环纠错**：中间步骤（如检测框或分割掩码）出错时，错误会直接传播到最终结果，系统无法自我诊断并重试。
    3.  **经验无法复用**：系统对重复出现的物体缺乏记忆，每次推理都从零开始。
*   **研究动机**：将该任务重新定义为“受限预算下的证据寻求”过程，通过动态编排技能和验证机制来提升鲁棒性和效率。

#### 2. 方法论
*   **核心思想**：构建一个闭环运行时（Runtime）框架，通过验证器（Verifier）对中间证据进行门控，并由路由器（Router）根据预算动态选择技能。
*   **关键技术组件**：
    *   **证据存储（Evidence Store）**：记录所有异构技能的输出（框、掩码、文本），并带有“来源追踪（Provenance）”，记录产生该证据的技能 ID、ROI 和缩放级别。
    *   **两层记忆系统**：
        *   **常识库（Mcs）**：存储离线收集的物体交互先验。
        *   **情节记忆（Mtt）**：在线积累验证成功的执行轨迹，供未来相似任务参考。
    *   **验证器（Verifier）——核心门控**：使用三个相对诊断信号：
        1.  **跨工具一致性 ($\omega$)**：不同技能（如检测与分割）是否指向同一区域。
        2.  **跨尺度稳定性 ($\zeta$)**：在缩放或重新裁剪后，预测区域是否保持稳定。
        3.  **证据充分性 ($\mu$)**：累积证据的加权可靠性是否达到决策阈值。
    *   **路由器（Router）**：基于启发式或 LLM 决策，选择预期收益（验证分增量）与成本比最高的下一个技能（如 Zoom-in、Web 搜索、交互想象）。
*   **算法流程**：输入查询 $\rightarrow$ 检索记忆 $\rightarrow$ 循环执行（技能调用 $\rightarrow$ 证据更新 $\rightarrow$ 验证诊断 $\rightarrow$ 针对性重试） $\rightarrow$ 达到阈值或预算耗尽 $\rightarrow$ 最终融合输出。

#### 6. 主要结论与发现
*   **性能领先**：A-Harness 在所有基准上均显著优于固定流水线模型，尤其在 ReasonAff 上达到了 69.68 gIoU。
*   **效率优化**：相比固定执行所有技能的系统，A-Harness 通过自适应路由减少了平均技能调用次数，降低了延迟和 API 成本。
*   **验证的价值**：验证器能有效拦截错误预测，触发针对性重试（如针对稳定性差的情况进行 Zoom-in），这是提升准确率的关键。
*   **记忆的红利**：情节记忆允许系统在处理重复类别时走“捷径”，进一步优化了成本-准确率的帕累托前沿。

#### 7. 优点
*   **闭环鲁棒性**：引入了类似人类“观察-验证-修正”的机制，解决了感知错误传播的问题。
*   **高度模块化**：技能（检测、分割、搜索）是可插拔的，能够随基础模型的进步而自动升级。
*   **预算感知**：显式考虑推理成本，使其在实际部署中比盲目堆叠模型的 Agent 更具实用性。

#### 8. 不足与局限
*   **依赖底层模型**：系统的上限仍受限于底层 LMM 的推理能力和感知模型的精度。
*   **物理反馈缺失**：目前的验证仅基于视觉和语义的一致性，缺乏真实物理环境中的交互反馈（如机器人实际抓取是否成功）。
*   **长尾场景挑战**：虽然有 Web 搜索技能，但对于极端罕见或违反常识的工业/特种物体，其验证逻辑可能失效。
