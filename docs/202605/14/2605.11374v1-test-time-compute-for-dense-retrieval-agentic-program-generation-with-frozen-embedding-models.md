## 摘要
普遍认为测试时计算（Test-time compute）仅对大型推理模型有益。我们证明它同样能提升小型嵌入模型的效果。大多数现代嵌入模型检查点都是从大型语言模型（LLM）骨干网络蒸馏而来，并继承了它们的表示空间；因此，冻结的嵌入模型应当能在无需重新训练的情况下，从额外的推理计算中获益。通过使用智能体程序搜索循环，我们在九十代演化中针对冻结的嵌入 API 探索了 259 个候选推理程序。整个帕累托前沿（Pareto frontier）最终收敛于一个单一的代数公式：局部前 K 个文档的 Softmax 加权质心与查询向量的插值。这种无参数的默认方案在跨越十倍参数范围的七个嵌入模型系列中显著提升了 nDCG@10 指标，且在留出的完整 BEIR 验证集上，所有测试模型均确认了这一提升。

## Abstract
Test-time compute is widely believed to benefit only large reasoning models. We show it also helps small embedding models. Most modern embedding checkpoints are distilled from large LLM backbones and inherit their representation space; a frozen embedding model should therefore benefit from extra inference compute without retraining. Using an agentic program-search loop, we explore 259 candidate inference programs over a frozen embedding API across ninety generations. The entire Pareto frontier collapses onto a single algebra: a softmax-weighted centroid of the local top-K documents interpolated with the query. This parameter-free default lifts nDCG@10 statistically significantly across seven embedding-model families spanning a tenfold parameter range, with held-out full-BEIR validation confirming the lift on every model tested.

论文探讨的核心问题是：**测试时计算（Test-time compute, TTC）是否能提升冻结的小型嵌入模型（Embedding Models）的检索性能？**
传统观点认为，通过增加推理时计算量来换取性能提升（如 Best-of-N 采样、思维链搜索）是大型推理模型（LRMs）的专利。然而，现代嵌入模型（如 jina-v5, e5, BGE）多由大型 LLM 蒸馏或初始化而来，继承了 LLM 的表示空间。作者假设这些模型也潜藏着通过额外推理计算提升性能的空间。研究旨在不改变模型参数、不引入外部生成模型的前提下，寻找最优的推理程序。

#### 2. 方法论：核心思想与技术细节
作者采用了一种**智能体程序搜索循环（Agentic Program-search Loop）**：
*   **核心思想**：利用 LLM（如 Claude 3.5 Opus）作为“提议者”，在冻结的嵌入 API 上编写 Python 程序，通过不断迭代搜索最优的推理逻辑。
*   **关键技术细节**：
    *   **搜索空间**：涵盖了文本操作、条件分支、分数空间融合、LoRA 融合、随机采样等 17 个子领域。
    *   **发现的最优代数（SOFTCENTROID）**：搜索最终收敛于一种简单的代数结构——**伪相关反馈（PRF）的变体**。即：先进行初步检索，获取 Top-K 文档，计算这些文档嵌入向量的 Softmax 加权质心，然后将其与原始查询向量进行插值，最后用更新后的查询向量重新排序。
    *   **公式逻辑**：更新后的查询 $q' = \text{normalize}((1-\alpha)q + \alpha \sum w_i d_i)$，其中权重 $w_i$ 通过对初始相似度进行温度系数为 $\tau$ 的 Softmax 计算得出。
    *   **课程账本（Lesson Ledger）**：记录失败的尝试（如随机集成、非对称 LoRA 编码等），防止智能体重复错误路径。

#### 6. 主要结论与发现
*   **TTC 有效性**：冻结的小型嵌入模型确实能从 TTC 中获益，且无需重新训练。
*   **帕累托前沿收敛**：复杂的程序（如随机集成）效果不佳，性能提升主要集中在“质心代数”类程序上。
*   **任务差异性**：在**对称检索**（如 ArguAna，查询和文档分布一致）中提升巨大（高达 +10 nDCG@10）；在医疗/科学检索中稳步提升；在问答类任务中需结合 BM25 分数融合才能获得显著收益。
*   **扩展性限制**：稠密检索的 TTC 收益呈凹函数，在 2× 到 4× 编码成本处迅速饱和，不像生成式 LLM 那样具有对数线性扩展性。

#### 7. 优点
*   **方法简洁且无参数**：最终发现的 `SOFTCENTROID` 方案不需要训练，易于在现有生产系统中部署。
*   **自动化发现**：展示了使用 LLM 智能体自动发现算法原语的潜力，而非依赖人工调优。
*   **鲁棒性强**：方案在多个模型家族和 ANN（如 HNSW）索引下均表现稳定。

#### 8. 不足与局限
*   **几何天花板**：受限于单向量余弦空间的几何特性，TTC 的提升存在上限，无法通过无限堆叠计算量来持续获取收益。
*   **任务局限**：在非对称的问答任务（NQ, HotpotQA）上，纯向量质心更新效果有限，必须依赖词法信号（BM25）补偿。
