## 摘要
大语言模型（LLMs）通常以固定参数部署，其性能往往通过在推理时分配更多计算资源来提升。虽然这种测试时缩放（test-time scaling）可能有效，但它无法纠正模型的错误认知，也无法使模型适应单个查询的特定结构。测试时优化通过在推理期间启用参数更新来解决这一局限性，但现有方法要么依赖外部数据，要么优化缺乏查询特定对齐的通用自监督目标。在这项工作中，我们提出了查询条件化测试时自我训练（QueST），这是一个在推理过程中利用直接源自输入查询的监督信号来调整模型参数的框架。我们的核心见解是，输入查询本身编码了足以构建结构相关“问题-解决方案”对的潜在信号。基于此，QueST 生成此类查询条件化的配对，并将其作为测试时参数高效微调的监督信号。随后，调整后的模型被用于生成最终答案，从而在无需任何外部数据的情况下实现查询特定的自适应。在七个数学推理基准测试和 GPQA-Diamond 科学推理基准测试中，QueST 一贯优于强大的测试时优化基准方法。这些结果表明，查询条件化自我训练是 LLM 测试时自适应的一种有效且实用的范式。

## Abstract
Large language models (LLMs) are typically deployed with fixed parameters, and their performance is often improved by allocating more computation at inference time. While such test-time scaling can be effective, it cannot correct model misconceptions or adapt the model to the specific structure of an individual query. Test-time optimization addresses this limitation by enabling parameter updates during inference, but existing approaches either rely on external data or optimize generic self-supervised objectives that lack query-specific alignment. In this work, we propose Query-Conditioned Test-Time Self-Training (QueST), a framework that adapts model parameters during inference using supervision derived directly from the input query. Our key insight is that the input query itself encodes latent signals sufficient for constructing structurally related problem--solution pairs. Based on this, QueST generates such query-conditioned pairs and uses them as supervision for parameter-efficient fine-tuning at test time. The adapted model is then used to produce the final answer, enabling query-specific adaptation without any external data. Across seven mathematical reasoning benchmarks and the GPQA-Diamond scientific reasoning benchmark, QueST consistently outperforms strong test-time optimization baselines. These results demonstrate that query-conditioned self-training is an effective and practical paradigm for test-time adaptation in LLMs.

大语言模型（LLM）在部署后参数通常是固定的。虽然可以通过“测试时缩放”（Test-time Scaling，如增加采样数或思维链长度）提升性能，但这种方法无法纠正模型内在的错误认知，也无法针对特定查询（Query）的独特结构进行参数级别的适配。
现有的“测试时优化”（Test-time Optimization, TTO）方法存在局限：
*   **依赖外部数据：** 如 TTT-NN 需要检索外部数据库。
*   **目标不匹配：** 如 TLM 最小化输入困惑度，但这种自监督目标与复杂的推理结构缺乏对齐。
**研究动机：** 探索一种无需外部数据、能直接利用输入查询信号在推理时动态调整模型参数的实用框架。

### 2. 论文提出的方法论：QueST
**核心思想：** 输入查询本身编码了足够的潜在信号，可以作为“种子”生成结构相关的辅助问题。通过在这些生成的辅助任务上进行即时自我训练，模型能“预热”并适配该查询所需的推理模式。

**关键技术细节与流程：**
1.  **查询条件化监督生成（Supervision Generation）：** 给定查询 $q$，使用模型自身作为生成器，产生 $N$ 个在概念、假设和推理模式上与 $q$ 相似的辅助“问题-答案”对 $D(q) = \{(x_i, y_i)\}_{i=1}^N$。
2.  **低秩测试时优化（Low-rank TTO）：** 
    *   为了保证效率和稳定性，不进行全参数微调，而是采用 **LoRA**（低秩自适应）。
    *   在 $D(q)$ 上进行 $T$ 步监督微调（SFT），最小化交叉熵损失：$\mathcal{L}_{TTO}(\theta; q) = \frac{1}{N} \sum \ell(f_\theta(x_i), y_i)$。
3.  **最终推理：** 使用更新后的参数 $\theta'$ 对原始查询 $q$ 生成最终响应 $y^*$。

### 7. 优点
*   **自给自足：** 无需外部知识库或人工标注数据，完全依赖输入查询。
*   **查询特定性：** 实现了真正的“因题制宜”，动态调整模型参数以匹配当前问题的逻辑结构。
*   **参数高效：** 结合 LoRA 使得测试时更新在计算上变得可行且稳定。

### 8. 不足与局限
*   **依赖初始查询质量：** 如果输入查询本身含糊不清或存在错误假设，生成的辅助监督信号可能具有误导性。
*   **计算开销：** 尽管 Token 效率高，但每条查询都需要进行梯度反向传播和参数更新，这在极高并发的生产环境中可能存在延迟挑战。
*   **缺乏持续性：** 目前模型参数在每条查询后会重置，无法实现跨查询的知识累积（持续学习）。
*   **错误传播风险：** 如果模型生成的辅助答案 $y_i$ 是错误的，可能会在微调阶段强化模型的错误认知（尽管实验表明其正面影响占主导）。
