## 摘要
普遍认为测试时计算仅对大型推理模型有益。我们证明它同样能提升小型嵌入模型。由于现代嵌入模型是从大语言模型（LLM）骨干网络蒸馏而来的，因此冻结的编码器应当能在无需重新训练的情况下，从额外的推理计算中获益。通过使用代理式程序搜索循环，我们在九十代演化中针对冻结的嵌入 API 探索了 259 个候选推理程序。整个帕累托前沿最终收敛于一个单一的代数公式：局部前 K 个文档的 Softmax 加权质心与查询的插值。这一默认方案不引入任何可训练参数，在涵盖十倍参数范围的七个嵌入模型家族中显著提升了 nDCG@10 指标，且在留出的全量 BEIR 验证中，所有测试模型均确认了这一提升。

## Abstract
Test-time compute is widely believed to benefit only large reasoning models. We show it also helps small embedding models. Since modern embedding models are distilled from LLM backbones, a frozen encoder should benefit from extra inference compute without retraining. Using an agentic program-search loop, we explore 259 candidate inference programs over a frozen embedding API across ninety generations. The entire Pareto frontier collapses onto a single algebra: a softmax-weighted centroid of the local top-K documents interpolated with the query. This default, which introduces no trainable parameters, lifts nDCG@10 statistically significantly across seven embedding-model families spanning a tenfold parameter range, with held-out full-BEIR validation confirming the lift on every model tested.

#### 2. 方法论
*   **核心思想**：构建一个**代理式程序搜索循环（Agentic Program-Search Loop）**。利用 LLM 智能体自动编写 Python 程序，在不改变嵌入模型参数的前提下，探索各种推理侧的操作组合。
*   **关键技术细节**：
    *   **搜索机制**：智能体在 90 代演化中生成了 259 个候选程序，涵盖了随机集成、LoRA 融合、分数空间融合等 17 个子领域。
    *   **核心算法（SOFTCENTROID）**：搜索最终收敛于一种简洁的代数形式。其流程为：
        1.  使用原始查询 $q$ 进行初步检索，获取 Top-K 文档。
        2.  计算这些文档嵌入向量的 **Softmax 加权质心**（权重由相似度和温度参数 $\tau$ 决定）。
        3.  将原始查询向量与该质心进行**线性插值**（插值系数为 $\alpha$），生成更新后的查询向量 $q'$。
        4.  使用 $q'$ 进行二次重排序。
*   **算法流程**：编码查询 -> 初步检索 -> 质心聚合 -> 向量插值 -> 最终重排序。

#### 6. 主要结论与发现
*   **TTC 有效性**：增加推理计算能显著提升冻结嵌入模型的性能，且存在帕累托前沿。
*   **几何特性**：嵌入空间的 TTC 规律与 LLM 不同。LLM 倾向于随机采样集成，而嵌入模型倾向于**结构化的几何更新**（向文档流形靠拢）。

*   **跨模型迁移**：发现的 `SOFTCENTROID` 方案具有极强的鲁棒性，无需针对特定任务调参即可在 7 个模型家族上取得正向收益。

#### 7. 优点
*   **零训练成本**：无需微调模型，直接应用于现有的冻结 API，极易部署。
*   **方法论创新**：首次将“智能体程序搜索”引入稠密检索的推理优化，自动发现了优于人工设计的代数方案。
*   **理论洞察**：明确了嵌入模型 TTC 的几何边界，即“质心更新”优于“随机集成”。

#### 8. 不足与局限
*   **任务依赖性**：该方法在对称检索（查询和文档分布一致）中效果极佳，但在事实性问答（Factoid QA，如 NQ 任务）中提升较小，甚至可能因质心偏移导致精度下降（需配合文中提出的事实性路由方案）。
*   **召回依赖**：如果初始检索的 Top-K 文档中完全不包含相关信息，质心更新可能会引入噪声。
