## 摘要
大语言模型（LLMs）展现出强大的推理能力，但其性能在分布偏移下往往会下降。现有的测试时自适应（TTA）方法依赖于基于梯度的更新，这需要白盒访问权限并产生巨大的开销，而无需训练的替代方案要么是静态的，要么依赖于外部引导。在本文中，我们提出了免训练测试时对比学习（TF-TTCL），这是一种无需训练的自适应框架，使冻结的 LLM 能够通过从自身的推理经验中蒸馏监督信号来实现在线改进。具体而言，TF-TTCL 通过三个核心模块实现了一个动态的“探索-反思-引导”（Explore-Reflect-Steer）循环：1）语义查询增强首先通过多智能体角色扮演使问题视角多样化，以生成不同的推理轨迹；2）对比经验蒸馏随后捕捉优劣轨迹之间的语义差异，并将其蒸馏为显式的文本规则；3）上下文规则检索最后在推理过程中激活这些存储的规则，动态地引导冻结的 LLM 走向鲁棒的推理模式，同时避免已观察到的错误。在闭口推理任务和开口评估任务上的广泛实验表明，在在线评估下，TF-TTCL 一贯优于强大的零样本基准和代表性的 TTA 方法。代码可在 https://github.com/KevinSCUTer/TF-TTCL 获取。

## Abstract
Large language models (LLMs) demonstrate strong reasoning capabilities, but their performance often degrades under distribution shift. Existing test-time adaptation (TTA) methods rely on gradient-based updates that require white-box access and need substantial overhead, while training-free alternatives are either static or depend on external guidance. In this paper, we propose Training-Free Test-Time Contrastive Learning TF-TTCL, a training-free adaptation framework that enables a frozen LLM to improve online by distilling supervision from its own inference experiences. Specifically, TF-TTCL implements a dynamic "Explore-Reflect-Steer" loop through three core modules: 1) Semantic Query Augmentation first diversifies problem views via multi-agent role-playing to generate different reasoning trajectories; 2) Contrastive Experience Distillation then captures the semantic gap between superior and inferior trajectories, distilling them into explicit textual rules; and 3) Contextual Rule Retrieval finally activates these stored rules during inference to dynamically steer the frozen LLM toward robust reasoning patterns while avoiding observed errors. Extensive experiments on closed-ended reasoning tasks and open-ended evaluation tasks demonstrate that TF-TTCL consistently outperforms strong zero-shot baselines and representative TTA methods under online evaluation. Code is available at https://github.com/KevinSCUTer/TF-TTCL.

#### 1. 核心问题与整体含义（研究动机和背景）
大语言模型（LLMs）虽然推理能力强大，但在面对**分布偏移（Distribution Shift）**或复杂的未见任务时，性能往往显著下降。现有的测试时自适应（TTA）方法存在两大局限：
*   **依赖梯度更新：** 大多数方法（如 Tent, TLM）需要访问模型内部参数（白盒），且在推理阶段产生巨大的计算和内存开销，不适用于通过 API 访问的黑盒模型。
*   **依赖外部引导：** 现有的免训练方法通常需要外部知识库或地面真值验证器（如单元测试），在现实部署场景中难以获得。

**研究动机：** 模仿人类从错误中反思的学习过程，提出一种无需梯度更新、无需外部反馈、仅通过模型自身推理经验进行在线自我进化的框架。

#### 2. 方法论：核心思想与关键技术
TF-TTCL 提出了一个动态的**“探索-反思-引导”（Explore-Reflect-Steer）**循环，包含三个核心模块：

*   **语义查询增强 (SQA - 探索)：**
    *   利用多智能体角色扮演（Teacher, Tutor, Student）模拟对比学习的数据增强。
    *   **Tutor** 将原始查询改写为多个语义等价但风格不同的变体，**Student** 针对这些变体生成多样化的推理路径，从而暴露模型在不同表达下的不确定性。
*   **对比经验蒸馏 (CED - 反思)：**
    *   **候选划分：** 根据一致性（闭口任务）或语义相似度（开口任务）将生成的多个回答划分为正样本集和负样本集。
    *   **样本选择：** 利用**序列级困惑度（PPL）**作为置信度指标，从正负集合中分别挑选出模型最“自信”的正确和错误样本。
    *   **规则蒸馏：** 调用模型对比优劣轨迹，提炼出显式的文本规则：**正向规则**（该做什么）和**负向规则**（避开什么，即“语义梯度”）。
*   **上下文规则检索 (CRR - 引导)：**
    *   维护一个动态更新的经验规则库。对于新输入的查询，通过向量相似度检索最相关的 Top-K 条正负规则，将其注入 Prompt 中，引导模型在不改变权重的情况下优化推理逻辑。

#### 6. 主要结论与发现
*   **性能提升：** TF-TTCL 在所有数学推理和领域自适应任务中均显著优于零样本和现有的 TTA 方法。
*   **负向规则更关键：** 实验发现，告诉模型“不要做什么”（负向规则）比告诉它“该做什么”（正向规则）对性能的提升更显著，因为负向规则提供了有效的纠错信号。
*   **黑盒适用性：** 在 DeepSeek-V3.2 和 Qwen-Plus 等顶级 API 模型上，TF-TTCL 依然能进一步提升其推理上限。
*   **内存管理：** 引入基于相似度的 FIFO 剪枝策略可以有效控制规则库的无限增长，甚至能通过过滤冗余信息略微提升精度。

#### 7. 优点（亮点）
*   **完全免训练：** 无需梯度，无需白盒权限，极大地扩展了测试时自适应的应用场景（如 API 调用）。
*   **无需外部反馈：** 仅靠模型自身的“反思”产生监督信号，解决了现实中缺乏 Ground Truth 的难题。
*   **双向引导：** 同时利用正向和负向经验规则，比单纯的 RAG 或 CoT 更具动态纠错能力。
*   **系统优化：** 考虑了推理延迟和内存爆炸问题，提出了并行化和剪枝方案，具备实际部署的可行性。

#### 8. 不足与局限
*   **性能饱和：** 随着模型规模增大或任务接近模型能力天花板，该方法的边际收益会递减。
*   **上下文窗口限制：** 尽管有剪枝策略，但一次性注入大量规则仍可能受限于模型的上下文窗口长度。
*   **探索效率：** 依赖于 Tutor 模型的改写能力，如果模型本身对某一领域完全无知，SQA 模块可能无法生成有效的对比样本。
*   **对抗风险：** 灵活的输入配置可能增加模型对对抗性 Prompt 注入的敏感性。
