Abstract 

Recent studies show that using potential out-of-distribution (OOD) labels from large corpora as auxiliary informa-tion can improve OOD detection in vision-language mod-els (VLMs). However, these methods often fail when real-world OOD samples fall outside the predefined OOD la-bel set. To address this limitation, we propose DynProto, a novel approach that learns OOD prototypes dynamically during testing using only in-distribution (ID) information. DynProto is inspired by a key observation: OOD samples predicted as the same ID class tend to cluster in the feature space. With this insight, we leverage easy-to-detect OOD samples as “anchors” to find their harder-to-detect, similar counterparts. To this end, DynProto introduces two mod-ules: Coarse OOD Pattern Capturing Module caches OOD patterns that are easily confused with each ID class during testing, and Fine-grained OOD Pattern Refinement Mod-ule subsequently clusters these patterns within each cache and aggregates them into representative OOD prototypes. By measuring similarity to ID and dynamic OOD proto-types, DynProto enables accurate OOD detection. Dyn-Proto significantly outperforms prior methods across multi-ple benchmarks. On ImageNet OOD benchmark, DynProto reduces FPR95 by 11.60% and improves AUROC by 4.70%. Moreover, the framework is architecture-agnostic and can be integrated into various backbones.

The key idea originates from an interesting observation: we found that some OOD samples predicted as the same ID class share highly similar visual features, forming compact clusters in the feature space. Although these OOD sam-ples are similar, the model can only correctly detect part of them, while the rest are still predicted as ID samples with high confidence, leading to severe misclassification issues. Based on this observation, we aim to dynamically col-lect easily identifiable OOD samples during testing and use them to generate OOD prototypes that attract other hard-to-detect but similar OOD samples. To this end, we design two modules: the Coarse OOD Pattern Capturing Module ,which maintains a per-class cache to dynamically collect potential OOD visual features predicted as that class with low confidence during testing, and the Fine-grained OOD Pattern Refinement Module , which clusters cached samples and aggregates features within each cluster into an OOD prototype for fine-grained OOD pattern representation. In addition, we construct ID prototypes from the training data. The final decision for a test sample is based on the similarity between the visual features of the test sample and the ID and OOD prototypes. As shown in Figure 1, compared to meth-ods that rely on external textual information, DynProto can accurately capture the true OOD distribution during testing. DynProto offers several distinct advantages. It does not rely on any external prior information and requires no addi-tional model training. Moreover, methods based on external textual labels are limited to VLMs, whereas our method is architecture-agnostic—it can be applied to both unimodal and multimodal architectures. Furthermore, DynProto can be built upon various OOD detection baselines, leveraging them as base detectors. The stronger the base detector, the higher the quality of the generated OOD prototypes. Our main contributions can be summarized as follows: • We reveal a key phenomenon that some OOD samples predicted as the same ID class share similar visual fea-tures, forming clusters in the feature space. Based on this, we propose to use easily detected OOD samples as an-chors to attract similar and hard-to-detect OOD samples. • We propose DynProto, a training-free, architecture-agnostic framework that dynamically learns OOD proto-types during inference. • Extensive experiments demonstrate that DynProto consis-tently outperforms SOTA methods, reducing FPR95 by 11.60% and improving AUROC by 4.70% on the Ima-geNet OOD benchmark with CLIP-B/16. 

2. Related Work 

Prototype Learning. A prototype is a representative feature embedding of a class, used to compute similar-ity between test features and the prototype for classifica-tion or representation learning. Recently, many studies have introduced prototype learning into OOD detection. SIREN [11] shapes hyperspherical feature representations with von Mises–Fisher prototypes to enhance separabil-ity between ID and OOD samples. CIDER [32] improves OOD detection by enforcing compact intra-class and dis-persed inter-class hyperspherical embeddings around learn-able class prototypes. PALM [27] models each class with a mixture of prototypes to better capture intra-class diversity. DHE [57] uses hyperspherical embeddings to ensure prov-able discrimination between ID and OOD samples by opti-mizing class prototypes on the sphere. SPROD [56] refines ID prototypes to mitigate the impact of spurious correla-tions. While these methods improve OOD performance by learning more representative ID prototypes, little attention has been paid to capture meaningful representations of true OOD samples. While POP [14] learns virtual OOD pro-totypes to reshape the decision boundary between ID and OOD data, these prototypes can not accurately capture the distribution of true OOD data. Our method fully leverages OOD data observed during testing, using them as anchors to capture other OOD patterns with similar characteristics. 

OOD Detection Based on External Textual Information. 

Benefiting from CLIP’s [37] strong generalization capa-bility in open-world scenarios, many recent methods have leveraged its rich textual knowledge by obtaining potential OOD text labels from external text corpora or large lan-guage models to assist CLIP in OOD detection. ZOC [12] trains a captioner on external datasets to generate potential OOD text labels. NegLabel [19] incorporates the Word-Net [29] to mine semantically unrelated words to the ID labels. CSP [5] enhances the activation probability of labels by constructing a set of low-correlation super-class labels. NegRefine [2] refines negative-label-based zero-shot OOD detection by filtering semantically ambiguous labels and in-troducing a multi-matching-aware scoring strategy. Syn-OOD [21] employs a diffusion-based in-painting pipeline to generate near-boundary OOD images, and then finetunes the CLIP model by aligning those synthetic samples with negative text-label features drawn from an external label space beyond the ID classes. 

Test-Time OOD Detection. Recently, growing attention has been devoted to utilizing information from real test-time scenarios to enhance OOD detection. AUTO [46] con-ducts test-time adaptation by directly optimizing the model via stochastic gradient descent, with the objective of reduc-ing overconfidence on potential OOD samples. RTL++ [13] models a linear relationship between OOD scores and fea-tures, but this assumption fails when the feature spaces of near OOD and ID samples overlap. AdaND [4] keeps the backbone model fixed and trains a lightweight noise detec-tor to identify and filter out OOD samples. TULIP [52] in-corporates prior knowledge of ID and OOD distributions during test time to calibrate the model’s predictive uncer-tainty. 3.3. Coarse OOD Pattern Capturing 

To construct OOD prototypes, we first need to collect di-verse OOD patterns. We maintain a cache for each ID class, which is initially empty. Given an input x, we feed it into detector and get an OOD score. If the score of x is lower, it is more likely to be an OOD sample. Therefore, if the score of x is lower than a predefined threshold, x is regarded as a potential OOD sample and its feature is stored in the cache associated with the predicted ID class. To maintain efficiency, each cache is assigned a fixed ca-pacity and updated using a first-in-first-out (FIFO) strategy. When the cache is not full, the visual feature v of an OOD candidate is directly stored in the cache of its predicted class. If the cache is full, the oldest entry is removed be-fore adding the new one. Based on temporal locality, recent samples are more likely to be correlated with the current test sample. Therefore, the FIFO strategy helps retain sam-ples most relevant to the current testing environment while discarding outdated ones, enhancing cache efficiency. 

3.4. Fine-grained OOD Pattern Refinement 

After cache updating, we use the collected samples to con-struct OOD prototypes. For any non-empty cache, we per-form BIRCH clustering [50] on the stored image features to identify different OOD patterns associated with the same ID class. Samples within the same cluster are aggregated by averaging to form a single OOD prototype. Similarly, ID training samples are aggregated by class to obtain one ID prototype per class. We denote the prototypes as P =[p1, p2, ..., pC , pC+1 , ..., pC+M ], where p1, ..., pC are ID prototypes, and pC+1 , ..., pC+M are constructed OOD pro-totypes. C is the number of ID classes, and M is the number of constructed OOD prototypes. Given a test image feature v = F(x) ∈ RD , we use the NegLabel score (Eq. 3) to measure whether the sample belongs to ID. Inspired by FA [28], we find that multiply-ing the OOD prototype term in the denominator of Snl by a coefficient K further improves the model’s OOD detec-tion performance. 3.5. Cold Start 

During the first Tcold iterations of testing, when the caches contain only a few samples, we use the MSP score [17] as the base OOD detector for unimodal models, while for VLMs we adopt the MCM score [30] to determine which test samples should be cached. We also experiment with other methods as base detectors, and detailed analyses are provided in Section 4.2. The score threshold for sample col-lection is denoted by θ. Since the score distribution of OOD samples is unavailable at test time, θ is estimated as the β-th percentile of the OOD score distribution derived from ID training data, with β = 5 in our experiments. As testing proceeds and more samples are accumulated, we stop using base detector and instead use DynProto score for selecting potential OOD samples, as DynProto score begins to outperform the base detector. 4. Experiments 

4.1. Experimental Setup 

Datasets. We perform extensive experiments using both the CIFAR-100 [20] and ImageNet [8] the as ID datasets. In line with prior work [23, 30], we select SVHN [35], LSUN-R [43], LSUN-C, iSUN [36], Texture [6], and Places365 [54] as the OOD datasets for CIFAR-100. For ImageNet, we use iNaturalist [40], SUN [44], Places [54], and Textures [6] as far-OOD datasets, while NINCO [3] and SSB-hard [41] are used as near-OOD datasets. We also ver-ify the effectiveness of DynProto on three more ID datasets: CIFAR10 [20], ImageNet100 [30], EuroSAT [16], and the experimental results are provided in the Appendix B. 

Implementation Details. We implement our method in Py-Torch and conduct all experiments on a NVIDIA GeForce RTX 3090 GPU with a batch size of 512. Following the test-time adaptation protocol, we mix and randomly shuffle ID and OOD samples. Each experiment is performed using five distinct random seeds for shuffling. The cache size is set to m = 30 for each class. The threshold θ is defined as the β = 5 th percentile of the OOD score distribution on the ID training set. The cold-start period Tcold is set to 5, and the coefficient K is set to 5. 

Metric. We use two evaluation metrics: the false positve rate at 95% true positive rate (FPR95) and the area under the receiver operating characteristic curve (AUROC). 

Baselines. We adopt a comprehensive set of competitive baselines for comparison. For Vision-Model-based OOD detection, we follow standard practice and use DenseNet-101 [18] and ResNet-50 [15] as backbones for CIFAR-100 and ImageNet, respectively. The compared methods include MSP [17], Energy [24], ReAct [39], DICE [38], VIM [42], ASH-S [9], LINe [1], OptFs [53], OODD [47], and CADRef [23]. For VLM-based OOD detection, we use CLIP-B/16 as the backbone and compare against MCM [30], CoOp [55], LoCoOp [34], NegLabel [19], CSP [5], AdaNeg [51], AdaND [4], NegRefine [2], FA [28], and SynOOD [21] as baselines. Among them, OODD, AdaNeg and AdaND are test-time adaptation methods. All baselines are evaluated using their original hyperparameter configurations to ensure fair comparison. 

4.2. Experimental Results 

Performance on CIFAR100 benchmarks. Table 1 sum-marizes the experimental results on the CIFAR-100 bench-mark. As shown, DynProto achieves SOTA performance on both DenseNet101 and CLIP-B/16 backbones. Notably, methods that rely on external textual information perform poorly on the SVHN dataset, where most other methods achieve strong results. This degradation arises because SVHN consists of digit images, while such text-based meth-ods lack semantically relevant tokens in their predefined la-bel sets. In contrast, our approach adaptively captures the feature distribution of these digit samples during testing, thereby constructing robust OOD prototypes and achieving near-zero detection errors.

These Gaussian noise samples have lower OOD scores, which help reduce the value of the dynamic threshold α during its calculation. This, in turn, prevents more ID samples from being stored in the cache when the number of ID samples significantly exceeds the OOD samples.

Hyperparameter Sensitivity Analysis. As shown in Fig-ure 4, we conduct a series of hyperparameter sensitivity ex-periments on ImageNet OOD benchmark to evaluate how different hyperparameters influence the performance. We first analyze the impact of the cache capacity m. A larger cache consistently leads to better performance, as it enables the cache to store more diverse OOD patterns. Next, we investigate the effect of β, which determines the OOD score threshold for sample collection during the cold-start stage. In our experiments, β ranges from 0 to 100, and we evaluate values between 5 and 85. As illus-trated, increasing β initially enhances model performance, which subsequently declines when the parameters continue to grow. This trend can be explained as follows. When β

is small, only a limited number of OOD samples are ad-mitted into the cache. As β increases, more OOD samples are incorporated, improving the model’s ability to discrim-inate between ID and OOD samples. However, once β ex-ceed certain thresholds, the number of cached OOD sam-ples reaches saturation, and additional increases cause more ID samples to be mistakenly stored, leading to cache con-tamination and a slightly drop in performance. We then analyze the effect of the scaling factor K in Eq. (4) which can be formulated as S(v) = AA+K·B , where 

A and B represent the overall similarities to ID and OOD prototypes, respectively, and K controls the relative weight of the OOD term. When K increases, the denominator 

A + K · B becomes larger, leading to a smaller score. For ID samples ( A ≫ B), moderate increases in K have little effect, but overly large values break the A ≫ B assump-tion and reduce ID scores. For OOD samples ( B > A ), increasing K further suppresses the score, but as K grows, the reduction becomes marginal since the score approaches zero. In practice, the optimal range for K is between 5 and 10, while values greater than 10 cause performance degra-dation due to excessive suppression of ID scores. 6. Conclusion 

We introduced DynProto, a test-time OOD detection frame-work that dynamically constructs OOD prototypes during inference, without requiring additional training or prede-fined OOD labels. DynProto exploits the tendency of OOD samples to cluster in the feature space. It uses easily de-tectable OOD instances as anchors and identifies harder cases through class-wise caching and clustering-based pro-totype aggregation. Extensive experiments across a range of OOD benchmarks demonstrate the effectiveness and ro-bustness of our approach. The core idea behind DynProto is general and could be extended to broader vision tasks, which will be explored in future work. Acknowledgements

A. Details of Datasets 

A.1. ID Datasets 

ImageNet [8] serves as one of the most widely used bench-marks for large-scale image classification. It contains ap-proximately 1.2 million training samples and 50,000 vali-dation samples distributed over 1,000 semantic categories. The dataset gathers images from a wide variety of natu-ral scenes, encompassing diverse object domains such as animals, plants, vehicles, and daily-use items. In our ex-periments, we adopt the 50,000-image validation portion of ImageNet-1K as the ID dataset. 

CIFAR10 and CIFAR100 [20] are classic benchmark datasets extensively used in computer vision and machine learning research, both derived from the 80 Million Tiny Images collection. CIFAR-10 contains 60,000 color images of size 32 × 32 , evenly distributed across 10 general object categories—such as airplanes, cats, dogs, and trucks—with 6,000 images per category, including 50,000 for training and 10,000 for testing. CIFAR-100, on the other hand, re-fines this setup into 100 fine-grained categories organized under 20 higher-level superclasses, while maintaining the same total image count and resolution (500 training and 100 testing images per class). 

A.2. OOD Dataset for Small-scale Dataset 

In our experiments under the small-scale dataset setting, we consider CIFAR-10 and CIFAR-100 as the ID datasets, and use SVHN [35], LSUN-R, LSUN-C [43], iSUN [36], Texture [6] and Places365 [54] as the corresponding OOD datasets. 

SVHN is a large-scale dataset derived from real-world street imagery, where individual digit instances are cropped from photographs of house numbers captured by Google Street View. Unlike synthetic digit datasets, SVHN con-tains complex visual variability, including diverse lighting conditions, cluttered urban backgrounds, and irregular digit placements, making it closer to natural image distributions. In our experiments, the entire SVHN test set, consisting of 26,032 images, is utilized as an OOD dataset for perfor-mance evaluation. 

LSUN is a large-scale benchmark for scene classification, encompassing a wide range of indoor and outdoor en-vironments, including categories such as bedrooms, liv-ing rooms, classrooms, and various building types. Two commonly used variants, LSUN C and LSUN R, are con-structed by applying random cropping and resizing transfor-mations to the original LSUN images, respectively. In our experiments, we sample 10000 images from each variant to form the OOD datasets used for evaluation. 

Places365 is a comprehensive large-scale dataset for scene recognition, comprising images categorized into 365 dis-tinct scene types that span a wide variety of indoor and out-door environments. The official test set contains 900 sam-ples per category. In this study, we construct an evaluation subset by randomly sampling 100 images from each class, resulting in a diverse and balanced collection used to assess model performance across different scene contexts. 

Textures is a dataset focused on recognizing material ap-pearance and surface properties, containing 5640 texture images categorized into 47 perceptually defined classes. These classes represent diverse patterns and materials, such as fabric, wood, and metal. Since there is no semantic over-lap between the Texture dataset and CIFAR classes, we uti-lize the entire dataset as OOD data in our experiments. 

iSUN is a large-scale dataset developed for saliency and visual attention research, containing 20608 natural scene images originally sourced from the SUN database. The dataset includes human eye-tracking annotations, capturing where people tend to focus when viewing complex real-world scenes. 8925 images are selected from iSUN that have no semantic overlap with the CIFAR dataset, using them as OOD samples to evaluate model robustness on vi-sually diverse and natural imagery. 

A.3. OOD Dataset for Large-scale Dataset 

In the large-scale dataset setting, we employ ImageNet as the ID datasets. For OOD evaluation, we adopt iNatural-ist [40], SUN [44], Places [54], and Textures as far-OOD datasets, while NINCO [3] and SSB-hard [41] are used as near-OOD datasets. 

iNaturalist is a large-scale dataset of natural world im-agery, encompassing 13 superclasses and 5,089 fine-grained categories that span various biological domains such as plants, insects, birds, and mammals. In this study, we utilize a curated subset provided by MCM [30], which includes 110 plant categories carefully selected to ensure no seman-tic overlap with ImageNet classes. This subset contains 10000 images. 

Places is a large-scale dataset of scene-centric images, cov-ering a wide range of semantic environments categorized into three broad domains: indoor, natural, and urban scenes. For our experiments, we employ a subset curated by MCM, which excludes any classes overlapping with ImageNet. This curated subset contains 10000 images. 

SUN is a comprehensive scene recognition dataset contain-ing 397 well-defined categories commonly used for bench-marking scene understanding models, and a total of 899 classes covering a wide range of indoor, urban, and natu-ral environments, both with and without human presence. In this study, we utilize a subset curated by MCM, which removes any overlapping categories with ImageNet. The resulting subset includes 10,000 images. 

Textures is a dataset comprising 5640 images of textures and abstract patterns, organized into 47 classes based on human perceptual similarity. As none of its classes overlap with ImageNet, the entire dataset was used as OOD dataset. 

NINCO is a purpose-built dataset for OOD detection, de-veloped to overcome the inherent limitations of ImageNet benchmarks. It includes 64 OOD categories with a total of 5879 images, each manually verified to ensure zero se-mantic overlap with any ImageNet class. The images were collected from a combination of existing datasets and newly sourced web data, followed by meticulous cleaning to guar-antee the complete absence of ID objects, making it a reli-able benchmark for assessing OOD detection performance. 

SSB-hard represents the most challenging split of the Se-mantic Shift Benchmark (SSB), created to rigorously assess OOD detection performance under subtle semantic varia-tions. It contains 49000 images drawn from 980 ImageNet-21K classes that were explicitly excluded from ImageNet. These classes are both visually and semantically close to the in-distribution categories, making SSB-hard a demand-ing and reliable benchmark for evaluating hard-OOD detec-tion scenarios.

D. More Analysis 

D.1. Impact of Cache Update Strategies 

In our current implementation, we adopt a FIFO policy for cache updates, where the oldest entry is discarded once the cache is full and a new OOD candidate is added. Further-more, following OODD [47], we also evaluate an alterna-tive strategy that replaces the cache entry with the highest OOD score when inserting new samples (RH). Considering that OOD types may exhibit abrupt changes in real-world scenarios, we simulate a situation where different OOD datasets appear sequentially over time. For instance, if the order of OOD datasets is SUN → Places → Textures, it is denoted as S → P → T . We simulate four such sequences: 

S → P → T , T → S → P and P → S → T . Addi-tionally, we also test a scenario where all OOD datasets are mixed with the ID data and shuffled, denoted as “All”. As shown in Table 15, our FIFO strategy consistently outperforms the RH-based cache update strategy under dif-ferent temporal drift scenarios, and also surpasses the base-line method NegLabel [19] that relies on external textual labels. According to the principle of temporal locality, the current test sample is more likely to be correlated with its re-cent samples. Therefore, adopting the FIFO strategy allows the model to better capture the evolving OOD in real time. In contrast, although the RH algorithm maintains a cache containing many representative OOD patterns, as testing progresses, the cache becomes dominated by earlier OOD samples with relatively low scores that are rarely replaced. Consequently, new OOD patterns fail to enter the cache, leading to reduced cache circulation and degraded adapt-ability to distributional drift. Moreover, in realistic OOD detection scenarios, various OOD samples often appear in a mixed and unpredictable manner. Even under such con-ditions, our model consistently outperforms existing base-lines. This demonstrates that our method can accurately capture the dynamic evolution of OOD distributions, main-taining strong robustness and adaptability in complex test-ing environments. D.2. Computation Cost. 

DynProto introduces a modest computational overhead dur-ing the caching and clustering stages, which we quantify through empirical measurements. With CLIP-B/16 as the backbone, standard CLIP inference runs at 453.29 FPS, whereas our method reaches 428.67 FPS, accompanied by an additional memory cost of 22 MB. Using ResNet-50 as the backbone, the standard inference speed is 841.43 FPS, while our approach achieves 811.90 FPS with an extra memory overhead of 60 MB.
