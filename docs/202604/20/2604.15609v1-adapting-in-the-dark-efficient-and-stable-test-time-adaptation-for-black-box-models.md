## 摘要
仅能通过 API 访问的黑盒模型的测试时自适应（TTA）仍是一个很大程度上尚未探索的挑战。现有的方法（如事后输出细化）提供的自适应能力有限，而零阶优化（ZOO）虽然支持输入空间自适应，但在无监督 TTA 设置下由于高昂的查询成本和优化难题而面临困境。我们提出了 BETA（黑盒高效测试时自适应），这是一个通过采用轻量级本地白盒引导模型来创建可计算梯度路径的框架，从而解决了上述局限性。通过预测协调技术结合一致性正则化和面向提示学习的过滤，BETA 实现了稳定的自适应，且无需额外的 API 调用，其延迟相对于标准推理几乎可以忽略不计。在 ImageNet-C 数据集上，BETA 在 ViT-B/16 上实现了 +7.1% 的准确率提升，在 CLIP 上提升了 +3.4%，超越了包括 TENT 和 TPT 在内的强力白盒及灰盒方法。在商业 API 上，BETA 在保持实时推理速度的同时，以低 250 倍的成本实现了与 ZOO 相当的性能，证明了其作为现实世界黑盒 TTA 实用且高效的解决方案的地位。

## Abstract
Test-Time Adaptation (TTA) for black-box models accessible only via APIs remains a largely unexplored challenge. Existing approaches such as post-hoc output refinement offer limited adaptive capacity, while Zeroth-Order Optimization (ZOO) enables input-space adaptation but faces high query costs and optimization challenges in the unsupervised TTA setting. We introduce BETA (Black-box Efficient Test-time Adaptation), a framework that addresses these limitations by employing a lightweight, local white-box steering model to create a tractable gradient pathway. Through a prediction harmonization technique combined with consistency regularization and prompt learning-oriented filtering, BETA enables stable adaptation with no additional API calls and negligible latency beyond standard inference. On ImageNet-C, BETA achieves a +7.1% accuracy gain on ViT-B/16 and +3.4% on CLIP, surpassing strong white-box and gray-box methods including TENT and TPT. On a commercial API, BETA achieves comparable performance to ZOO at 250x lower cost while maintaining real-time inference speed, establishing it as a practical and efficient solution for real-world black-box TTA.

#### 1. 核心问题与整体含义（研究动机和背景）

*   **核心挑战**：许多 SOTA 模型（如 GPT-4o, CLIP）仅通过 API 提供黑盒访问，用户无法获取参数、梯度或中间特征。
*   **现有局限**：
    *   **输出细化（如 LAME）**：仅调整预测结果，自适应能力有限。
    *   **零阶优化（ZOO）**：通过多次 API 调用估算梯度，成本极高且在无监督场景下极不稳定，容易导致性能崩溃。
    *   **灰盒方法**：需要访问中间特征或 Token，不符合严格的黑盒定义。
*   **研究动机**：开发一种既能实现输入空间自适应（如视觉提示学习），又具备高查询效率（单次 API 调用）且优化稳定的黑盒 TTA 框架。

#### 2. 方法论
论文提出了 **BETA (Black-box Efficient Test-time Adaptation)** 框架，其核心思想是利用一个**本地轻量级白盒“引导模型”（Steering Model）**来辅助黑盒目标的自适应。

*   **核心思想**：在客户端运行一个小的白盒模型（如 ViT-Small），通过它建立可计算的梯度路径，从而优化输入端的“视觉提示”（Visual Prompt）。
*   **关键技术细节**：
    1.  **预测协调（Prediction Harmonization）**：将本地引导模型 $f_S$ 和黑盒目标模型 $f_B$ 的输出概率进行加权融合，构建一个共享的优化目标。
    2.  **非对称优化路径**：虽然目标是优化融合后的分布，但梯度仅通过本地白盒模型回传。通过数学证明，这种方式引入了一个 Jensen–Shannon (JS) 散度项，强制本地模型与黑盒模型在预测上保持一致，从而起到“锚定”作用。
    3.  **稳定性机制**：
        *   **一致性正则化**：通过 KL 散度约束原始图像与带提示图像的预测一致性，防止提示学习破坏语义。
        *   **面向提示的过滤**：仅选择低熵（高置信度）样本更新提示，过滤掉噪声干扰。
    4.  **联合优化**：同时更新视觉提示（影响输入）和本地引导模型的归一化层参数。
