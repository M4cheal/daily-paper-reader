## 摘要
尽管由带有可验证奖励的强化学习（RLVR）驱动的大语言推理模型（LRMs）取得了显著进展，但在监督信号极其昂贵或无法获取的专业或新兴领域，这一范式受到了根本性的限制，这为测试时自适应带来了关键挑战。虽然现有的测试时方法提供了一种潜在的解决方案，但它们受限于从静态查询集中学习，存在对文本模式产生过拟合的风险。为了填补这一空白，我们引入了测试时变分合成（TTVS），这是一个新颖的框架，通过从无标签测试查询中动态增强训练流，使 LRMs 能够实现自我进化。TTVS 包含两个协同模块：(1) 在线变分合成，它将静态测试查询转化为多样化、语义等价变体的动态流，迫使模型学习底层问题逻辑而非表面模式；(2) 测试时混合探索，它在合成变体之间平衡了准确率驱动的利用与一致性驱动的探索。大量实验表明，TTVS 在八种模型架构上均表现出优异的性能。值得注意的是，仅使用无标签的测试时数据，TTVS 不仅超越了其他测试时自适应方法，还优于在海量高质量有标签数据上训练的最先进的基于监督强化学习的技术。

## Abstract
Despite significant advances in Large Reasoning Models (LRMs) driven by reinforcement learning with verifiable rewards (RLVR), this paradigm is fundamentally limited in specialized or novel domains where such supervision is prohibitively expensive or unavailable, posing a key challenge for test-time adaptation. While existing test-time methods offer a potential solution, they are constrained by learning from static query sets, risking overfitting to textual patterns. To address this gap, we introduce Test-Time Variational Synthesis (TTVS), a novel framework that enables LRMs to self-evolve by dynamically augmenting the training stream from unlabeled test queries. TTVS comprises two synergistic modules: (1) Online Variational Synthesis, which transforms static test queries into a dynamic stream of diverse, semantically-equivalent variations, enforcing the model to learn underlying problem logic rather than superficial patterns; (2) Test-time Hybrid Exploration, which balances accuracy-driven exploitation with consistency-driven exploration across synthetic variants. Extensive experiments show TTVS yields superior performance across eight model architectures. Notably, using only unlabeled test-time data, TTVS not only surpasses other test-time adaptation methods but also outperforms state-of-the-art supervised RL-based techniques trained on vast, high-quality labeled data.

### 论文总结：TTVS (Test-time Variational Synthesis)

#### 1. 核心问题与整体含义（研究动机和背景）
论文针对大语言推理模型（LRMs）在专业或新兴领域（如医疗诊断、航空航天）面临的**监督信号缺失**问题展开研究。虽然基于可验证奖励的强化学习（RLVR）效果显著，但其高度依赖高质量的标注数据。现有的“测试时自适应”（Test-time Adaptation）方法虽然能利用无标签数据，但通常局限于**静态查询集**，容易导致模型过拟合于特定的文本模式而非底层的逻辑推理。
**核心目标**：提出一种无需外部监督、能让模型在测试阶段通过动态合成数据实现“自我进化”的强化学习框架。

#### 2. 方法论：核心思想与关键技术
TTVS 框架由两个协同模块组成，旨在将静态测试任务转化为动态的训练流：
*   **在线变分合成 (Online Variational Synthesis)**：
    1.  **伪标签生成**：对原始查询进行多次采样（Rollouts），通过多数投票（Majority Voting）确定共识答案作为伪标签。
    2.  **变分增强**：利用模型自身将原始查询改写为多个语义等价但表述不同的变体。
    3.  **在线过滤**：根据模型对原始问题的准确率（难度过滤）和生成文本的长度（质量过滤）筛选出适合训练的样本，确保模型关注“跳一跳够得着”的问题。
*   **测试时混合探索 (Test-time Hybrid Exploration)**：
    1.  **组内探索 (IGE)**：针对每个具体查询（原始或合成）进行独立的策略优化，驱动模型在特定表述下追求准确性。
    2.  **跨组探索 (CGE)**：将原始查询及其变体的采样结果混合，进行统一的多数投票并计算奖励。这迫使模型在不同表述间保持逻辑一致性，学习不变的推理逻辑。
*   **优化算法**：该框架与具体的 RL 算法无关，实验中主要采用了 GRPO（组相对策略优化），但也证明了其在 OPO 和 DAPO 上的适用性。

#### 3. 实验设计
*   **数据集**：涵盖了数学推理和科学问答的多个基准，包括 MATH-500、AIME-2024、AMC-2023 和 GPQA。此外还扩展到了 DeepMath-103K（防污染测试）、MedMCQA（医疗领域）和 OpenBookQA（常识推理）。
*   **模型架构**：在 8 种不同规模的模型上进行了验证，包括 Qwen3 (1.7B/4B/8B)、Qwen2.5 (1.5B/3B/7B) 和 LLaMA (3.2-3B/3.1-8B)。
*   **对比方法 (Baselines)**：
    1.  **有监督 RL 模型**：如 DeepSeek-R1-Distill、OpenReasoner-Zero 等（使用了数十万条标注数据）。
    2.  **测试时 RL 方法**：TTRL（仅在原始无标签数据上进行 RL，无变分合成）。

#### 4. 资源与算力
*   **硬件环境**：所有实验均在 **8 台 NVIDIA H800 80GB GPU** 上进行。
*   **训练细节**：优化器使用 AdamW，学习率峰值为 $5 \times 10^{-7}$。采样阶段每条查询生成 32 个响应进行投票。

#### 5. 实验数量与充分性
*   **实验规模**：论文进行了非常详尽的实验，涵盖了 8 种模型架构和 7 个以上的基准测试集。
*   **消融实验**：针对 IGE/CGE 模块的有效性、不同 RL 算法的兼容性、计算成本、问题难度等级、采样数量（Rollouts）等多个维度进行了深入分析。
*   **客观性与公平性**：通过在完全未见过的 DeepMath-103K 数据集上测试，排除了数据污染的嫌疑；通过与使用海量标注数据的模型对比，突出了该方法在无监督场景下的优越性。实验设计严谨，对比维度全面。

#### 6. 主要结论与发现
*   **性能卓越**：TTVS 在所有测试架构上均显著优于 TTRL。
*   **超越有监督模型**：令人惊讶的是，仅使用无标签测试数据的 TTVS（如 Qwen2.5-7B 规模），在某些指标上超过了在 80 万条标注数据上训练的 DeepSeek-R1-Distill-7B。
*   **鲁棒性强**：随着问题难度增加（如 MATH-500 的 L5 级别），TTVS 相比基准方法的领先优势更加扩大，证明变分合成有助于填补知识空白。
*   **高效性**：在相同的计算预算（采样数）下，TTVS 的显存占用与 TTRL 相当，但推理出的答案更简洁（Token 长度更短）。

#### 7. 优点
*   **动态增强**：打破了静态测试集的限制，通过自我改写实现了数据的“无限”扩充。
*   **双重约束**：通过 IGE 保证准确性，通过 CGE 保证逻辑一致性，有效缓解了对表面文本模式的过拟合。
*   **通用性**：不依赖特定模型架构或 RL 算法，具有很强的迁移潜力。

#### 8. 不足与局限
*   **模型规模限制**：受限于算力，实验主要在 8B 及以下规模模型上进行，尚未在 32B 或更大的模型上验证扩展性。
*   **模态局限**：目前仅限于纯文本任务，未探索在多模态（如视觉-语言）任务中的表现。
*   **依赖初始能力**：对于完全没有推理能力的基座模型，可能需要极少量的专家数据进行“热身”（Warmup）才能开启自我进化循环。
*   **可验证性要求**：目前主要适用于有明确客观答案（如数学、代码）的领域，对于主观性强的任务（如创意写作）仍需引入“模型即裁判”（LM-as-a-Judge）机制。
