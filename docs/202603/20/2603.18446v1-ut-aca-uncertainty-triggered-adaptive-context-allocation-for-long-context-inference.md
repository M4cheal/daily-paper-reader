## 摘要
由于注意力稀释和分布外退化，长上下文推理对于大语言模型而言仍具挑战性。上下文选择通过关注键值缓存（KV cache）条目的子集来缓解这一限制，但尽管 token 级别的上下文需求高度不均匀，大多数方法在整个解码过程中仍分配固定的上下文预算。为了解决这一问题，我们提出了不确定性触发的自适应上下文分配（UT-ACA），这是一个在推理阶段根据逐 token 不确定性动态调整上下文窗口的框架。UT-ACA 学习了一个不确定性检测器，该检测器结合了语义嵌入与基于 logit 的置信度，同时考虑了跨解码步骤的不确定性累积。当显示证据不足时，UT-ACA 会选择性地回退，扩大上下文窗口，并在额外支持下重新生成 token。实验表明，在长上下文设置下，UT-ACA 在保持生成质量的同时，大幅减少了平均上下文使用量。

## Abstract
Long-context inference remains challenging for large language models due to attention dilution and out-of-distribution degradation. Context selection mitigates this limitation by attending to a subset of key-value cache entries, yet most methods allocate a fixed context budget throughout decoding despite highly non-uniform token-level contextual demands. To address this issue, we propose Uncertainty-Triggered Adaptive Context Allocation (UT-ACA), an inference-time framework that dynamically adjusts the context window based on token-wise uncertainty. UT-ACA learns an uncertainty detector that combines semantic embeddings with logit-based confidence while accounting for uncertainty accumulation across decoding steps. When insufficient evidence is indicated, UT-ACA selectively rolls back, expands the context window, and regenerates the token with additional support. Experiments show that UT-ACA substantially reduces average context usage while preserving generation quality in long-context settings.

#### 1. 核心问题与整体含义（研究动机和背景）
大语言模型（LLM）在处理长文本（如长文档问答、多文档摘要）时，面临**注意力稀释**（无关信息干扰）和**分布外（OOD）退化**（推理长度超出预训练范围导致性能下降）的问题。现有的上下文管理方法（如 KV Cache 压缩或选择）通常在解码过程中分配**固定的上下文预算**。然而，不同 token 的生成难度并不一致：有些 token 仅依赖局部上下文，而有些则需要全局证据。固定预算会导致简单步骤的资源浪费和困难步骤的证据不足。本文旨在通过**动态、按需分配**上下文资源，在提升推理效率的同时保证生成质量。

#### 2. 论文提出的方法论：UT-ACA 框架
UT-ACA 是一种推理时（Inference-time）框架，其核心思想是“**先尝试、再检测、必要时回退重来**”。
*   **核心流程**：
    1.  **初步生成**：使用较小的上下文窗口（Compact Window）尝试生成一个 token。
    2.  **不确定性检测**：利用专门训练的**不确定性检测器**分析该 token 的输出概率（Logits）和内部语义特征（Hidden States）。
    3.  **生成难度评估（GDM）**：检测器输出三分类指标：正确（Correct）、未知（Unknown）或幻觉（Hallucination）。
    4.  **触发回退与扩展**：若检测到高不确定性（非正确类占主导），系统执行**回退（Rollback）**，丢弃该 token，**扩大上下文窗口**（检索更多 KV 块），然后重新生成。
*   **关键技术细节**：
    *   **双编码器融合**：结合 Logit Margin（最高与次高概率之差）和语义嵌入（来自模型最后一层注意力输出）。
    *   **时序建模**：使用 **LSTM** 捕捉解码过程中的不确定性累积效应。
    *   **自适应策略**：生成确定性 token 后，窗口会逐步缩小以节省资源。

#### 6. 主要结论与发现
*   **效率与质量的双赢**：UT-ACA 在显著减少平均上下文使用量（例如在某些任务中减少约 25%-50%）的同时，保持甚至提升了长文本生成的准确率。
*   **不确定性是有效信号**：Token 级的不确定性（尤其是结合了语义和概率信号后）能准确指示何时需要更多上下文。
*   **三分类的必要性**：区分“未知”和“幻觉”比简单的“对/错”二分类能提供更细致的监督信号，使检测器更稳定。

#### 7. 优点
*   **动态性**：打破了固定预算的限制，实现了真正的按需分配。
*   **即插即用**：作为推理时框架，不需要对原始 LLM 进行微调（仅需训练一个极轻量级的检测器）。
*   **鲁棒性**：通过回退机制纠正了因上下文不足导致的早期错误，防止错误累积。
*   **评估创新**：提出了“概念准确率（CAS）”，比传统的字符串匹配更能反映语义层面的正确性。

#### 8. 不足与局限
*   **延迟开销**：虽然减少了上下文计算量，但**回退和重新生成**机制增加了额外的推理步骤，导致每 token 的平均延迟（Latency）并未随 token 数减少而等比例下降，甚至略有增加。
*   **检测器依赖**：检测器的性能受限于训练数据的覆盖面，对于完全未见过的任务领域，其触发准确性可能下降。
*   **应用限制**：在对实时性要求极高的场景下，回退机制带来的延迟波动可能是一个挑战。
