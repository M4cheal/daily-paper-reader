## 摘要
基础模型正从离线预测器转型为预期在长周期内运行的部署系统。在实际部署中，目标并非固定不变：领域会发生漂移，用户偏好会演变，且在模型发布后会出现新任务。这使得持续学习和即时个性化从可选功能提升为核心架构需求。然而，大多数适配流水线仍遵循静态权重范式：在训练（或任何适配步骤）之后，推理过程无论用户意图、领域或特定实例约束如何，都执行单一的参数向量。这将训练或适配后的模型视为参数空间中的一个孤立点。在异构且持续演进的状态下，不同的目标可能在参数上诱导出分离的可行区域，迫使任何单一的共享更新陷入折中、干扰或过度专业化。因此，持续学习和个性化通常被实现为对共享权重的重复覆盖，从而面临先前学习行为退化的风险。我们提出了 HY-WU（权重释放），这是一种记忆优先的适配框架，它将适配压力从覆盖单一共享参数点转移开。HY-WU 将函数式（算子级）记忆实现为一个神经模块：一个根据实例条件即时合成权重更新的生成器，从而在无需测试时优化的情况下产生特定于实例的算子。

## Abstract
Foundation models are transitioning from offline predictors to deployed systems expected to operate over long time horizons. In real deployments, objectives are not fixed: domains drift, user preferences evolve, and new tasks appear after the model has shipped. This elevates continual learning and instant personalization from optional features to core architectural requirements. Yet most adaptation pipelines still follow a static weight paradigm: after training (or after any adaptation step), inference executes a single parameter vector regardless of user intent, domain, or instance-specific constraints. This treats the trained or adapted model as a single point in parameter space. In heterogeneous and continually evolving regimes, distinct objectives can induce separated feasible regions over parameters, forcing any single shared update into compromise, interference, or overspecialization. As a result, continual learning and personalization are often implemented as repeated overwriting of shared weights, risking degradation of previously learned behaviors. We propose HY-WU (Weight Unleashing), a memory-first adaptation framework that shifts adaptation pressure away from overwriting a single shared parameter point. HY-WU implements functional (operator-level) memory as a neural module: a generator that synthesizes weight updates on-the-fly from the instance condition, yielding instance-specific operators without test-time optimization.

### 2. 方法论：核心思想与关键技术
*   **核心思想**：将记忆实现为算子合成。给定输入 $x$（图像+指令），生成器 $g_\phi$ 预测对应的 LoRA 更新 $\Delta\theta(x)$，使推理时的算子变为 $f_{\theta+\Delta\theta(x)}$。
*   **关键技术细节**：
    1.  **端到端即时训练（On-the-fly Training）**：不同于以往超网络（Hypernetwork）需要预先收集大量微调后的 Checkpoint 进行重构训练，HY-WU 直接通过下游任务损失函数优化生成器，简化了流程并提升了扩展性。
    2.  **秩锚定 2D 参数标记化（Rank-anchored 2D Parameter Tokenization）**：针对大模型各层维度不一的问题，利用 LoRA 秩（Rank）固定的特性，将异构参数切分为统一形状的 Token，保留了权重的 2D 结构语义。
    3.  **神经网络 Transformer (NNT)**：采用 Transformer 架构作为生成器，并引入**因子化注意力（Factorized Attention）**（分为层内和层间注意力），以处理超长参数序列的计算压力。
    4.  **稳定性设计**：采用零初始化策略（LoRA 的 B 矩阵初始化为零），确保训练初期模型行为与原始冻结骨干一致。

### 6. 主要结论与发现
*   **性能领先**：HY-WU 在 GEdit-Bench 排名开源模型第一，在人类偏好测试中显著优于主流开源模型，并超越了部分强力闭源模型（如 GPT Image 1.5）。
*   **解决冲突**：静态适配在面对冲突目标时会产生“折中”行为，而 HY-WU 能保持清晰的定向专业化。
*   **参数空间结构化**：生成的权重在参数空间中并非杂乱无章，而是呈现出与输入语义高度一致的流形结构（通过 t-SNE 可视化证实）。
*   **正向缩放规律**：增加生成器容量或 LoRA 秩能持续提升性能，展现了“函数式存储”的 Scaling Law。

### 7. 优点
*   **范式创新**：从“优化单一参数点”转向“学习参数映射函数”，为持续学习提供了新视角。
*   **无需测试时优化**：相比于传统的个性化方法（如 DreamBooth 需要针对每个新对象微调），HY-WU 实现了即时的推理侧适配。
*   **架构通用性**：不依赖特定的扩散模型架构，可扩展至多种 Transformer 骨干。
*   **系统级优化**：针对长参数序列生成的工程实现（Tokenization + Factorized Attention）具有很强的实用参考价值。

### 8. 不足与局限
*   **计算开销**：虽然无需测试时微调，但推理时需要额外运行一个 8B 的生成器来合成权重，增加了首字延迟或初始计算负担。
*   **应用范围限制**：目前仅在图像编辑（Part I）中验证，尚未在长程对话、复杂智能体决策等更广阔的领域展示其“持续学习”的完整闭环。
*   **检索增强缺失**：论文承认目前未对比检索增强（RAG）类记忆方法，两者如何结合（事实性记忆 vs 过程性记忆）仍是开放课题。
