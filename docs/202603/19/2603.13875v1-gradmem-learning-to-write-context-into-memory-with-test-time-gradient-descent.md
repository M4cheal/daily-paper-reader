## 摘要
许多大语言模型应用需要基于长上下文进行条件生成。Transformer 通常通过存储过去激活的大型逐层 KV 缓存来支持这一点，这会产生巨大的内存开销。一个理想的替代方案是压缩内存：一次性读取上下文，将其存储在紧凑状态中，并根据该状态回答多个查询。我们在上下文移除设置下研究了这一点，即模型在推理时必须在无法访问原始上下文的情况下生成答案。我们引入了 GradMem，它通过逐样本的测试时优化将上下文写入内存。给定一个上下文，GradMem 在保持模型权重冻结的同时，对一小组前缀内存标记执行几步梯度下降。GradMem 显式地优化模型级的自监督上下文重建损失，从而实现具有迭代误差校正的损失驱动写入操作，这与仅前向传播的方法不同。在关联键值检索任务上，GradMem 在相同内存大小的情况下优于仅前向传播的内存写入器，且额外的梯度步骤比重复的前向写入能更有效地扩展容量。我们进一步证明了 GradMem 可以迁移到合成基准之外：配合预训练语言模型，它在包括 bAbI 和 SQuAD 变体在内的自然语言任务上取得了具有竞争力的结果，且仅依赖于编码在内存中的信息。

## Abstract
Many large language model applications require conditioning on long contexts. Transformers typically support this by storing a large per-layer KV-cache of past activations, which incurs substantial memory overhead. A desirable alternative is ompressive memory: read a context once, store it in a compact state, and answer many queries from that state. We study this in a context removal setting, where the model must generate an answer without access to the original context at inference time. We introduce GradMem, which writes context into memory via per-sample test-time optimization. Given a context, GradMem performs a few steps of gradient descent on a small set of prefix memory tokens while keeping model weights frozen. GradMem explicitly optimizes a model-level self-supervised context reconstruction loss, resulting in a loss-driven write operation with iterative error correction, unlike forward-only methods. On associative key--value retrieval, GradMem outperforms forward-only memory writers with the same memory size, and additional gradient steps scale capacity much more effectively than repeated forward writes. We further show that GradMem transfers beyond synthetic benchmarks: with pretrained language models, it attains competitive results on natural language tasks including bAbI and SQuAD variants, relying only on information encoded in memory.

#### 1. 核心问题与整体含义（研究动机和背景）
大语言模型（LLM）在处理长上下文时，主流的 **KV-cache（键值缓存）** 机制面临巨大的内存开销，且难以生成可移植的上下文表示。虽然现有的“压缩记忆”方法（如 RMT）尝试将上下文编码为固定大小的向量，但这些方法多采用**仅前向（Forward-only）**的写入规则，缺乏对写入质量的反馈和纠错能力。

本文提出了 **GradMem**，其核心动机是利用**测试时训练（Test-Time Training, TTT）**的思想，将记忆写入过程视为一个优化问题。通过在推理阶段对少量记忆令牌（Memory Tokens）进行梯度下降，模型能够迭代地修正压缩误差，从而在极小的内存占用下实现更高的信息存储容量。

#### 2. 方法论：核心思想与技术细节
GradMem 将推理过程分为两个阶段：**写入（WRITE）**和**读取（READ）**。

*   **核心思想**：不改变模型权重 $\theta$，而是将前缀记忆令牌 $M$ 视为可学习的参数。在给定上下文 $C$ 时，通过梯度下降优化 $M$，使其能够“重构”出上下文。
*   **写入阶段（WRITE）**：
    1.  初始化记忆 $M_0$（这是跨样本共享的元学习参数）。
    2.  定义自监督目标 $L_{write}$：即模型在给定 $M$ 的情况下，预测上下文 $C$ 中后续标记的交叉熵损失。
    3.  执行 $K$ 步梯度下降：$M_{k+1} = M_k - \alpha \nabla_{M_k} L_{write}$，得到最终记忆 $\hat{M}$。
*   **读取阶段（READ）**：
    1.  移除原始上下文 $C$。
    2.  将优化后的 $\hat{M}$ 与查询 $Q$ 拼接，模型预测目标答案 $Y$。
*   **关键技术：元学习（Meta-learning）**：
    在训练阶段，通过对写入过程的 $K$ 步优化进行**二阶导数回传**（类似于 MAML），训练模型权重 $\theta$ 和初始记忆 $M_0$，使得模型学会如何通过少量的梯度步数高效地存储关键信息。

#### 7. 优点
*   **迭代纠错**：相比一成不变的前向编码，梯度下降提供了闭环反馈，能处理更复杂的上下文。
*   **任务无关性**：写入目标是纯自监督的重构损失，不需要针对特定下游任务调整写入逻辑。
*   **计算灵活性**：用户可以根据推理预算灵活调整梯度步数 $K$。

#### 8. 不足与局限

*   **架构限制**：目前主要基于前缀记忆（Prefix Memory），对于需要极长程依赖的超长文档，固定数量的令牌可能仍存在瓶颈。
