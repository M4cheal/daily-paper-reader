## 摘要
现有的生成模型（如扩散模型和自回归网络）本质上是静态的，依赖于一组固定的预训练参数来处理所有输入。相比之下，人类能够根据每个感知或想象的情境灵活地调整其内部生成表征。受此启发，我们提出了 Composer，这是一种基于测试时实例特定参数组合的自适应生成建模新范式。Composer 在推理时生成基于输入的参数适配，并将其注入预训练模型的权重中，从而在无需微调或重新训练的情况下实现针对每个输入的专门化。适配在多步生成之前进行一次，以极低的计算和内存开销产生更高质量、上下文感知的输出。实验表明，Composer 在各种生成模型和用例中显著提升了性能，包括轻量级/量化模型和测试时缩放（test-time scaling）。通过利用输入感知的参数组合，Composer 为设计能够动态适应每个输入的生成模型建立了一种新范式，超越了静态参数化。

## Abstract
Existing generative models, such as diffusion and auto-regressive networks, are inherently static, relying on a fixed set of pretrained parameters to handle all inputs. In contrast, humans flexibly adapt their internal generative representations to each perceptual or imaginative context. Inspired by this capability, we introduce Composer, a new paradigm for adaptive generative modeling based on test-time instance-specific parameter composition. Composer generates input-conditioned parameter adaptations at inference time, which are injected into the pretrained model's weights, enabling per-input specialization without fine-tuning or retraining. Adaptation occurs once prior to multi-step generation, yielding higher-quality, context-aware outputs with minimal computational and memory overhead. Experiments show that Composer substantially improves performance across diverse generative models and use cases, including lightweight/quantized models and test-time scaling. By leveraging input-aware parameter composition, Composer establishes a new paradigm for designing generative models that dynamically adapt to each input, moving beyond static parameterization.

### 2. 方法论
*   **核心思想**：提出 **Composer** 范式，通过一个轻量级的辅助网络（Meta-generator），在推理阶段根据输入条件实时生成“低秩更新矩阵”，并将其注入预训练权重中。
*   **关键技术细节**：
    *   **参数组合公式**：将权重矩阵 $W$ 重新表述为 $W' = W + AB$，其中 $A$ 和 $B$ 是由 Composer 生成的低秩矩阵（秩 $r \ll d$）。

    *   **全局-局部注意力（Global-Local Attention）**：为了平衡计算效率和建模能力，设计了一种特殊的注意力机制，让组件 Token 关注提示 Token，同时在组件内部保持局部块注意力。
    *   **上下文感知训练策略**：在训练 Composer 时，通过控制 Batch 中语义相似和相异样本的比例（参数 $\alpha$），强制模型在相似输入下保持适配的一致性，在不同输入下保持区分度。

### 6. 主要结论与发现
*   **性能提升**：Composer 在所有测试的骨干网络上均一致地降低了 FID 并提升了 IS。例如在 VAR d-30 上，FID 从 1.97 降至 1.79。
*   **量化补偿**：在极低比特（如 2-bit）量化下，Composer 能显著修复由于精度损失导致的图像质量下降。
*   **高效适配**：相比于需要迭代优化的 TTT，Composer 仅需在生成循环开始前进行一次前向计算，几乎不影响推理速度。

### 7. 优点
*   **即插即用**：无需修改原始骨干网络的架构，也无需对骨干网络进行微调。
*   **极高效率**：通过低秩分解和一次性生成，解决了测试时适配的计算瓶颈。
*   **泛化性强**：同时适用于扩散模型和自回归模型，且能与量化、测试时缩放等技术完美兼容。

### 8. 不足与局限
*   **极速生成场景的占比**：虽然在多步生成（如 250 步 DiT）中开销可忽略不计，但在单步（1-step）生成模型中，Composer 生成参数的时间占比会变得明显。

*   **任务覆盖**：目前主要集中在图像生成，尚未在视频生成或更广泛的视觉迁移学习任务中进行大规模验证。
