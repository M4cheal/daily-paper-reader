## 摘要
我们引入了 PACE，这是一种无反向传播的持续测试时自适应系统，可直接优化归一化层的仿射参数。现有的无导数方法难以平衡运行效率与学习能力，因为它们要么将更新限制在输入提示（prompts）上，要么无论领域是否稳定都需要进行持续且耗费资源的自适应。为了解决这些局限性，PACE 利用协方差矩阵自适应进化策略（CMA-ES）结合 Fastfood 投影，在低维子空间内优化高维仿射参数，从而实现了卓越的自适应性能。此外，我们通过引入自适应停止准则和领域专用向量库来消除冗余计算，从而提高了运行效率。我们的框架在持续分布偏移下的多个基准测试中达到了最先进的准确率，与现有的无反向传播方法相比，运行时间缩短了 50% 以上。

## Abstract
We introduce PACE, a backpropagation-free continual test-time adaptation system that directly optimizes the affine parameters of normalization layers. Existing derivative-free approaches struggle to balance runtime efficiency with learning capacity, as they either restrict updates to input prompts or require continuous, resource-intensive adaptation regardless of domain stability. To address these limitations, PACE leverages the Covariance Matrix Adaptation Evolution Strategy with the Fastfood projection to optimize high-dimensional affine parameters within a low-dimensional subspace, leading to superior adaptive performance. Furthermore, we enhance the runtime efficiency by incorporating an adaptation stopping criterion and a domain-specialized vector bank to eliminate redundant computation. Our framework achieves state-of-the-art accuracy across multiple benchmarks under continual distribution shifts, reducing runtime by over 50% compared to existing backpropagation-free methods.

*   **核心问题**：现有的测试时自适应（TTA）方法主要分为基于反向传播（BP）和无反向传播（BP-free）两类。BP方法内存开销大且不支持量化模型；而现有的BP-free方法（如FOA、ZOA）面临**学习能力受限**（仅更新Prompt或噪声梯度估计）与**计算开销高**（每个样本都需要多次前向传播，且在领域稳定时仍持续计算）的矛盾。
*   **研究动机**：在资源受限的边缘设备上，需要一种既能有效更新模型权重（提升精度），又能动态停止计算（节省能耗），且无需反向传播的持续自适应系统。

### 2. 论文提出的方法论：PACE
PACE（Projected Adaptation via Covariance Evolution）框架包含三个核心组件：
*   **子空间自适应（Subspace Adaptation）**：
    *   **核心思想**：利用TTA梯度具有低内在维度的特性，在低维子空间内优化归一化层（Normalization layers）的仿射参数。
    *   **技术细节**：使用**CMA-ES（协方差矩阵自适应进化策略）**优化一个低维向量 $v$，通过 **Fastfood 变换**（一种高效的伪随机投影，利用快速沃尔什-哈达玛变换 FWHT）将其投影到高维参数空间。这种方式避免了存储巨大的投影矩阵，极大地节省了内存。
*   **自适应停止准则（Adaptation Stopping）**：
    *   **机制**：监控CMA-ES分布均值的相对变化。当变化低于阈值 $\epsilon$ 时，认为模型已收敛至当前领域的最优状态，停止自适应计算，后续推理仅需单次前向传播。
    *   **重启**：通过监测模型浅层激活统计量的KL散度来检测领域偏移（Domain Shift），一旦检测到新领域则重启自适应。
*   **领域专用向量库（Domain-Specialized Vector Bank）**：
    *   **知识复用**：维护一个小型内存库存储历史领域的优化向量。当检测到新领域时，从库中检索最匹配的向量作为初始化，以应对循环出现的领域。
