## 摘要
持续测试时适应（CTTA）在未标记的非平稳数据流上在线更新预训练模型，同时将其锚定到冻结的源检查点。仅当源可靠时，该锚点才有用。然而，在CCC-Hard上，ResNet-50源的top-1准确率降至约1.3%，而现有的源锚定CTTA方法继续应用相同的锚定强度。我们将这种失败模式称为盲目锚定，并提出了RMemSafe，这是ROID的可靠性门控扩展，它利用冻结源的归一化预测熵来衰减目标中所有显式源耦合使用。当源后验接近均匀时，门关闭：源锚点和一致性过滤器消失，目标简化为由ROID的基础损失加上边际校准组成的源无关后备方案。结合ASR，RMemSafe在9个匹配分割的持续损坏单元中的8个上实现了最低错误率，并且在所有9个单元上是最佳基于重置的方法，在ResNet-50上相比ROID+ASR提升了1.05个百分点，在ViT-B/16上提升了0.48个百分点。控制源退化扫描显示，其损害斜率比ROID+ASR浅1.13倍，与优雅退化预测一致。熵门检测的是高熵源崩溃，而非自信错误但低熵的源；该范围已被明确评估和讨论。

## Abstract
Continual test-time adaptation (CTTA) updates a pretrained model online on an unlabeled, non-stationary stream while anchoring it to a frozen source checkpoint. This anchor is useful only when the source remains reliable. On CCC-Hard, however, a ResNet-50 source falls to approximately $1.3\%$ top-$1$ accuracy, while existing source-anchored CTTA methods continue applying the same anchor strength. We call this failure mode blind anchoring and propose RMemSafe, a reliability-gated extension of ROID that uses the frozen source's normalized predictive entropy to attenuate all explicit source-coupled uses in the objective. When the source posterior approaches uniformity, the gate closes: the source anchor and agreement filter vanish, and the objective reduces to a source-agnostic fallback comprising ROID's base losses plus marginal calibration. Combined with ASR, RMemSafe achieves the lowest error on $8$ of $9$ matched-split continual-corruption cells and is the best reset-based method on all $9$, improving ROID+ASR by $1.05$~pp on ResNet-50 and $0.48$~pp on ViT-B/16. A controlled source-degradation sweep shows a $1.13{\times}$ shallower harm slope than ROID+ASR, consistent with the graceful-decay prediction. The entropy gate detects high-entropy source collapse, not confidently wrong low-entropy sources; this scope is explicitly evaluated and discussed.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：持续测试时适应（CTTA）在未标记、非平稳数据流上在线更新预训练模型，同时将其锚定到冻结的源模型检查点。现有方法假设源模型在整个适应过程中保持可靠。
- **核心问题**：在严重分布偏移（如CCC-Hard）下，源模型准确率可能骤降（ResNet-50仅有约1.3% top-1准确率），但现有CTTA方法仍以固定强度向这个已失效的源模型拉近，导致“盲目锚定”（blind anchoring）这一系统性失败模式。
- **研究动机**：需要一个运行时安全检查，当源模型失效时能自动衰减源耦合项，实现优雅退化（graceful decay），避免灾难性遗忘和性能崩溃。
- **整体含义**：首次明确诊断并命名了CTTA中的“盲目锚定”问题，并提出首个通过运行时可靠性信号（源熵）门控所有显式源耦合项的CTTA方法，具有解析的优雅退化保证。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
利用冻结源模型的归一化预测熵作为运行时可靠性信号，构造一个门控标量 `Rsrc = max(0, 1 - Hsrc)`，用于衰减所有显式依赖源模型的优化项（锚定损失、一致性过滤器、锚定内的散度项）。当源后验接近均匀（高熵）时，门关闭，目标退化为源无关的后备目标（ROID基础损失 + 边际校准）。

### 2.2 关键技术细节
- **可靠性门控（Reliability Gate）**：计算源模型的归一化熵 `Hsrc`，定义可靠性标量 `Rsrc = max(0, 1 - Hsrc)`。该标量范围[0,1]。
- **动态锚定损失**：将原本固定的ℓ2锚定强度替换为 `Lanch = λ · Rsrc · (1 + α*Hexp + β*DJS(psrc||pexp)) · ||θexp - θ*src||²`，其中外部的Rsrc使锚定仅当源可靠时才生效，内部项在专家不确定或偏离源时增强锚定。
- **源-专家一致性门控**：余弦一致性权重 `wcos = Rsrc * wraw + (1-Rsrc)·1`，当源不可靠时，一致性过滤器退化为全通（无过滤）。
- **后备稳定器**：
  - 边际校准（Marginal Calibration）：KL散度惩罚批次边缘与历史先验的差异，防止类别坍塌。
  - 置信度缩放学习率：`ηeff = η · [ηmin + (1-ηmin)(1-Hexp)]`，根据专家不确定性动态缩放学习率。
  - 解耦翻转插值推理：对原始和水平翻转视图的logits进行置信度自适应插值。
- **优雅退化性质（Proposition 1）**：当Hsrc→1（源完全混乱），Rsrc→0，则Lanch→0，wcos→1，总目标退化为 `Lslr + Lcons + λmarg Lmarg`，完全独立于源模型参数。
- **算法流程**（Algorithm 1）：每批次计算源和专家的预测分布，计算熵和可靠性标量，调制锚定和一致性权重，加上基础损失和边际校准，用置信度缩放学习率更新专家参数，结合ASR自适应重置控制器。

## 6. 主要结论与发现

1. **诊断“盲目锚定”**：在CCC-Hard上，源准确率~1%，但先前方法仍以全强度锚定，导致系统性失败。
2. **RMemSafe实现8/9基准单元最低错误**：在匹配分割比较中，RMemSafe+ASR在9个单元中的8个上取得最低错误，在全部9个上优于所有其他重置基方法。
3. **对ROID+ASR的改进**：CCC平均错误降低1.05 pp（ResNet-50）和0.48 pp（ViT-B/16）；长时间回访（CIN-C, IN-C）降低0.47-0.88 pp。
4. **优雅退化的实证**：控制源退化实验中，RMemSafe的损害斜率（11.43 pp/单位S）比ROID+ASR（12.92 pp/单位S）浅1.13倍，且差距随源退化程度单调增大。
5. **门控有效性**：可靠性标量Rsrc在CCC-Hard运行时约0.26，将锚定有效强度衰减3.8倍；超参数敏感性极低（跨16倍范围变化<0.21 pp）。
6. **重置范式在ViT-Hard上的失败**：首次在匹配分割粒度上证明所有重置基方法在ViT-Hard上均劣于非重置ROID，但RMemSafe在重置基方法中最佳。
